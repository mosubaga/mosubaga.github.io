Go Programming Fundamentals

This guide introduces the fundamental concepts of Go with practical, copy-friendly code samples. Start here for syntax, tooling, values, control flow, and collections before moving into the more focused topics in the navigation.

1. Introduction and Ecosystem

Go is an open-source language focused on simplicity and efficiency. To verify your installation, you can use the command line:

$ go version
# Example output: go version go1.26.4 darwin/arm64

You can also use the Go Playground at go.dev/play to run code directly in your browser.

2. The Go Toolchain

The Go command is a "Swiss Army Knife" for developers, handling everything from running scripts to building binaries.

# 1. Run code immediately without creating a visible binary
$ go run main.go

# 2. Build an executable binary for your OS
$ go build main.go

# 3. Format code according to standard rules
$ go fmt main.go

# 4. Initialize a new module for dependency management
$ go mod init myproject

3. Program Structure

All Go programs are organized into packages, with package main serving as the entry point for executable applications.

package main // Mandatory package declaration

import "fmt" // Import standard library package

// The main function is where execution starts
func main() {
    fmt.Println("Hello, Go!") // Formatted output
}

4. Variables and Constants

Go supports both explicit declarations and short variable declarations (which use type inference) inside functions.

package main
import "fmt"

const Pi = 3.14 // Constant declaration

func main() {
    // 1. Explicit declaration with type
    var name string = "Gopher"

    // 2. Short declaration (only inside functions)
    age := 10

    // 3. Declaration of multiple variables
    var x, y int = 1, 2

    // 4. Using Iota for auto-incrementing constants
    const (
        Zero = iota // 0
        One         // 1
        Two         // 2
    )

    fmt.Println(name, age, x, y, Pi, Zero, One, Two)
}

5. Primitive Data Types

Go provides fixed-size numeric types, booleans, and context-aware characters called runes.

var (
    a bool    = true    // Boolean
    b int     = 42      // Integer (32 or 64 bits depending on OS)
    c float64 = 19.99   // Floating-point
    d string  = "Go"    // Sequence of characters
    e rune    = '世'    // Single Unicode code point (int32 alias)
)

fmt.Println(a, b, c, d, e)

6. Operators and Conversions

Go does not perform implicit conversion; you must explicitly convert types to combine them in expressions.

kayak := 275      // int
soccerBall := 19.5 // float64

// Incorrect: kayak + soccerBall would cause a compiler error

// Correct: Explicitly convert int to float64
total := float64(kayak) + soccerBall

// Comparison operators produce a bool result
isExpensive := total > 100
fmt.Println(total, isExpensive)

7. Flow Control

Go uses if/else, switch, and for loops. Notably, for is the only looping construct available.

// 1. If statement with initialization
if x := 10; x > 5 {
    fmt.Println("x is large")
}

// 2. Switch statement (no parentheses required)
switch day := "Monday"; day {
case "Monday", "Tuesday":
    fmt.Println("Weekday")
default:
    fmt.Println("Other day")
}

// 3. For loop (Traditional, while-style, and range)
for i := 0; i < 5; i++ {
    fmt.Println(i)
}

nums := []int{2, 3, 4}
for index, value := range nums {
    fmt.Printf("Index %d has value %d\n", index, value)
}

8. Basic Collection Types

Collections allow grouping elements. Slices are dynamic and more commonly used than fixed-length arrays.

// 1. Array (Fixed length)
var names = [3]string{"Ann", "Bob", "Cal"}

// 2. Slice (Variable length)
items := []string{"Hat", "Shirt"} // Literal
items = append(items, "Boots")    // Dynamic growth

// 3. Map (Key-Value pairs)
prices := make(map[string]float64)
prices["Kayak"] = 279.0
prices["Paddle"] = 40.0

// Checking for existence in a map
_, exists := prices["Hat"] // exists is false; use _ to discard value

Note: All code examples above are compliant with standard Go rules. For example, variables declared must be used, or the compiler will report an error.
