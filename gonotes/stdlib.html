Leveraging the Go Standard Library

Go provides a powerful and comprehensive standard library that offers essential building blocks for modern applications, including networking, math, and data serialization. This guide focuses on three critical areas: time management, logging, and file I/O.

1. Time Management

The time package is the primary tool for representing moments in time and durations. It encapsulates everything required for dates and times, including time zone management and mathematical operations.

Key Functions and Types

  • time.Time: Represents a specific instant in time with nanosecond precision.
  • time.Now(): Returns the current local time.
  • time.Duration: Represents the elapsed time between two instants as an int64 value in nanoseconds.
package main
import (
    "fmt"
    "time"
)

func main() {
    // 1. Getting current time
    now := time.Now()
    
    // 2. Formatting (uses a unique reference time: Jan 2 15:04:05 MST 2006)
    fmt.Println(now.Format("2006-01-02 15:04:05"))

    // 3. Parsing strings into time
    layout := "2006-Jan-02"
    parsed, err := time.Parse(layout, "2023-Jun-09")
    if err != nil {
        panic(err)
    }

    // 4. Time arithmetic
    future := now.Add(10 * time.Minute)
    diff := future.Sub(now) // returns a Duration
    fmt.Printf("Parsed: %s; Difference: %v\n", parsed.Format(time.DateOnly), diff)
}

2. Logging

Logging is essential for troubleshooting and examining program behavior. Go offers two primary logging packages: the traditional log package and the newer structured log/slog package.

The Standard log Package

The log package sends messages to standard error by default and automatically includes a timestamp. It supports basic functions like Print, Fatal (which exits the program), and Panic.

import "log"

func main() {
    log.Println("Basic log message")
    
    // Customizing log output (e.g., adding line numbers)
    log.SetFlags(log.LstdFlags | log.Lshortfile)
}

Structured Logging with slog

Introduced in Go 1.21, log/slog provides structured logs, often in JSON format, which are easier for automated tools to process.

import (
    "log/slog"
    "os"
)

func main() {
    // Simple structured log
    slog.Info("usage statistics", "user", "gopher", "attempts", 3)

    // JSON output handler
    logger := slog.New(slog.NewJSONHandler(os.Stdout, nil))
    logger.Error("failed to connect", "retry", true)
}

3. Basic File I/O

File operations are primarily handled by the os package, while the io and bufio packages provide interfaces and buffering for efficient data transfer.

Reading Files

The simplest way to read a small file is os.ReadFile, which loads the entire content into memory. For larger files, use os.Open with a buffer or a scanner.

// 1. Quick read
data, err := os.ReadFile("config.json")
if err != nil {
    return err
}
fmt.Println(string(data))

// 2. Buffered read (line by line)
file, err := os.Open("large.txt")
if err != nil {
    return err
}
defer file.Close()
scanner := bufio.NewScanner(file)
for scanner.Scan() {
    fmt.Println(scanner.Text())
}
if err := scanner.Err(); err != nil {
    return err
}

Writing Files

os.WriteFile is used for one-shot writes, while os.Create or os.OpenFile (with flags like os.O_APPEND) allow for more control.

// 1. One-shot write
err := os.WriteFile("output.txt", []byte("Hello Go"), 0644)

// 2. Appending to a file
f, err := os.OpenFile("log.txt", os.O_APPEND|os.O_CREATE|os.O_WRONLY, 0644)
if err != nil {
    return err
}
defer f.Close()
if _, err := f.WriteString("New log entry\n"); err != nil {
    return err
}

Note: Always remember to call Close() on file descriptors to prevent resource leaks. Using the defer keyword is the idiomatic way to ensure a file closes when the function returns.
