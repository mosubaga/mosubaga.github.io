Network and Web Services in Go

Go is an ideal language for building massively scalable backend systems and RESTful APIs. The standard library package net/http provides the foundational layer for writing both web servers and clients with a focus on simplicity and reliability.

1. Developing HTTP Servers

A Go application typically receives connections directly from clients using its built-in web server, meaning no intermediate application server is required. The server logic is organized around handlers and multiplexers.

Basic Server with Handlers

A handler is a type that implements the ServeHTTP method, which is responsible for writing headers and bodies into the HTTP response. The simplest way to define endpoints is using http.HandleFunc.

package main
import (
    "fmt"
    "net/http"
)

// Define a handler function
func myHandler(w http.ResponseWriter, r *http.Request) {
    fmt.Fprintf(w, "Serving: %s\n", r.URL.Path)
}

func main() {
    // Register the handler for the root path
    http.HandleFunc("/", myHandler)
    
    // Start the server on port 8080
    err := http.ListenAndServe(":8080", nil)
    if err != nil {
        panic(err)
    }
}

Request Multiplexing (ServeMux)

The http.ServeMux acts as an HTTP request router, comparing incoming URIs against predefined patterns to execute the associated handler. A more specific pattern takes precedence over a more general one.

2. Developing HTTP Clients

Go's net/http package offers straightforward functions like http.Get() and http.NewRequest() for clients to interact with web services.

Simple GET Request

For quick requests where advanced configuration isn't needed, the shared DefaultClient can be used.

package main
import (
    "fmt"
    "io"
    "net/http"
)

func main() {
    url := "https://www.golang.org"
    response, err := http.Get(url)
    if err != nil {
        panic(err)
    }
    defer response.Body.Close() // Mandatory to prevent memory leaks

    body, err := io.ReadAll(response.Body)
    if err != nil {
        panic(err)
    }
    fmt.Println(string(body))
}

Advanced Client Requests

To apply timeouts or custom headers, you should create a custom http.Client and use http.NewRequest().

3. Handling JSON and XML Data

Exchanging structured data is fundamental to modern web services. Go provides the encoding/json and encoding/xml packages for serialization (marshaling) and deserialization (unmarshaling).

Working with JSON

JSON is the de facto standard for web APIs. Go uses struct tags to map between JSON keys and Go structure fields.

import (
    "encoding/json"
    "fmt"
)

type User struct {
    Name     string `json:"username"` // Maps "username" in JSON to "Name"
    Password string `json:"-"`        // Ignored by the encoder
}

func main() {
    u := User{Name: "Gopher"}
    // Marshal: Go struct -> JSON bytes
    jsonData, err := json.Marshal(&u)
    if err != nil {
        panic(err)
    }

    // Unmarshal: JSON bytes -> Go struct
    var u2 User
    if err := json.Unmarshal(jsonData, &u2); err != nil {
        panic(err)
    }
    fmt.Println(u2.Name) // Gopher
}

Working with XML

XML processing follows a similar pattern to JSON but supports more complex structures like attributes and namespaces.

import "encoding/xml"

type Person struct {
    XMLName xml.Name `xml:"person"`     // Root element name (xml.Name from encoding/xml)
    Type    string   `xml:"type,attr"`  // Map as an XML attribute
    Content string   `xml:",chardata"`  // Raw character data
}

4. Building REST APIs

REST (Representational State Transfer) is an architectural style designed for standardized data access. RESTful services primarily focus on defining Go structures and using marshaling/unmarshaling to exchange JSON data between clients and servers.

Standard REST Verbs

  • POST: Create new resources.
  • GET: Read existing resources.
  • PUT: Update existing resources with a full version.
  • DELETE: Remove existing resources.

Routing with go-chi

go-chi is a lightweight, idiomatic router that is fully compatible with net/http. It supports URL parameters, middleware chaining, and composable subrouters, making it the popular choice for production REST APIs.

// Install: go get github.com/go-chi/chi/v5
import (
    "fmt"
    "net/http"
    "github.com/go-chi/chi/v5"
    "github.com/go-chi/chi/v5/middleware"
)

func main() {
    r := chi.NewRouter()

    // Built-in middleware
    r.Use(middleware.Logger)
    r.Use(middleware.Recoverer)

    // Define routes by HTTP method
    r.Get("/articles", listArticlesHandler)
    r.Post("/articles", createArticleHandler)

    // Named URL parameters with a subrouter
    r.Route("/articles/{id}", func(r chi.Router) {
        r.Get("/", getArticleHandler)
        r.Put("/", updateArticleHandler)
        r.Delete("/", deleteArticleHandler)
    })

    if err := http.ListenAndServe(":1234", r); err != nil {
        panic(err)
    }
}

func listArticlesHandler(w http.ResponseWriter, r *http.Request) {
    fmt.Fprintln(w, "Listing articles")
}

func createArticleHandler(w http.ResponseWriter, r *http.Request) {
    fmt.Fprintln(w, "Creating article")
}

// Access a URL parameter inside any handler
func getArticleHandler(w http.ResponseWriter, r *http.Request) {
    id := chi.URLParam(r, "id")
    fmt.Fprintf(w, "Article ID: %s", id)
}

func updateArticleHandler(w http.ResponseWriter, r *http.Request) {
    id := chi.URLParam(r, "id")
    fmt.Fprintf(w, "Updating article %s", id)
}

func deleteArticleHandler(w http.ResponseWriter, r *http.Request) {
    id := chi.URLParam(r, "id")
    fmt.Fprintf(w, "Deleting article %s", id)
}

Security Tip: Always close the response.Body when finished to prevent the server from reserving resources indefinitely. For production services, always implement timeouts for both reading and writing processes.
