Intermediate Go: Core Concepts

This guide covers Pointers, Structs, Methods, and Interfaces. These features allow you to build complex data types and define common behaviors, forming the backbone of robust Go applications.

1. Pointers

A pointer is a variable that stores the memory address of another value rather than the value itself. Pointers are fundamental for sharing memory across different parts of a program without copying data.

Key Operators

  • & (Address Operator): Generates a pointer to a variable.
  • * (Indirection/Dereference Operator): Accesses the value stored at the address a pointer refers to.
package main
import "fmt"

func main() {
    value := 42
    ptr := &value // & creates a pointer to value

    fmt.Println("Address:", ptr)  // e.g., 0xc000012070
    fmt.Println("Value:", *ptr)   // 42 (Dereferencing)

    *ptr = 100 // Modifying the original value via the pointer
    fmt.Println("Updated Value:", value) // 100
}

Note: Go does not support pointer arithmetic (like incrementing a memory address to access the next array element), which prevents a common class of bugs found in C/C++.

2. Structs

Structs (short for structures) are user-defined types that group together related data fields into a single entity. They are essential for modeling real-world objects.

Declaration and Visibility

Fields starting with an uppercase letter are exported (public) and accessible outside the package; those starting with lowercase are unexported (private).

type Product struct {
    Name     string  // Exported field
    price    float64 // Unexported field
}

func main() {
    // 1. Using a struct literal (named fields are always preferred)
    p1 := Product{Name: "Kayak", price: 275.0}

    // 2. Creating a pointer to a struct
    p2 := &Product{Name: "Lifejacket", price: 48.95}

    // Accessing fields via dot notation
    fmt.Println(p1.Name, p2.Name) // Kayak Lifejacket
}

3. Methods

A method is a function attached to a specific data type using a receiver. Methods allow you to add behavior directly to your structs.

Value vs. Pointer Receivers

  • Value Receiver: Operates on a copy of the data. Changes do not affect the original.
  • Pointer Receiver: Operates on the original value via its memory address. Use this to modify the receiver's state.
type Rectangle struct {
    Length, Width float64
}

// Value receiver: Does not modify the original
func (r Rectangle) Area() float64 {
    return r.Length * r.Width
}

// Pointer receiver: Can modify the original
func (r *Rectangle) Scale(factor float64) {
    r.Length *= factor
    r.Width *= factor
}

func main() {
    rect := Rectangle{10, 5}
    rect.Scale(2) // Go automatically handles the pointer conversion
    fmt.Println("New Area:", rect.Area()) // 200
}

4. Interfaces

An interface is an abstract type that defines a contract of behavior by specifying a set of method signatures.

Implicit Implementation

Unlike Java or C#, Go uses implicit satisfaction. A type implements an interface simply by defining all required methods; there is no implements keyword.

// 1. Define the interface
type Speaker interface {
    Speak() string
}

type Dog struct{}
func (d Dog) Speak() string { return "Woof!" } // Dog satisfies Speaker

type Cat struct{}
func (c Cat) Speak() string { return "Meow!" } // Cat satisfies Speaker

func MakeItSpeak(s Speaker) {
    fmt.Println(s.Speak()) // Polymorphism: works for any Speaker
}

func main() {
    d := Dog{}
    c := Cat{}
    MakeItSpeak(d)
    MakeItSpeak(c)
}

The Empty Interface

Represented as interface{} (or any in Go 1.18+), the empty interface has no methods and can therefore hold a value of any type.

func PrintAnything(v any) { // 'any' is an alias for interface{}
    fmt.Println(v)
}

Best Practice Summary

  • Use Pointers to avoid expensive data copies or to allow functions to modify values.
  • Use Structs to group data; remember that field visibility is controlled by capitalization.
  • Use Pointer Receivers for methods if the struct is large or needs to be modified.
  • Use Interfaces to define shared behavior and enable flexible, decoupled code.
