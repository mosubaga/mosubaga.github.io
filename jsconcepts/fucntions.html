Callbacks and Higher-Order Functions

Callbacks and higher-order functions are two of the most important ideas in JavaScript. They explain how event handlers, array methods, asynchronous code, React props, middleware, and many web application patterns work.

1. Big Picture

In JavaScript, functions are values. This means a function can be stored in a variable, passed into another function, returned from another function, placed inside an array, or used as an object property. 

function greet(name) {
  return `Hello, ${name}`;
}

const fn = greet;

console.log(fn("Nobu")); // Hello, Nobu

Because functions are values, JavaScript can use callbacks and higher-order functions naturally.

2. What is a Callback Function?

A callback is a function passed into another function so it can be called later. The receiving function decides when to execute the callback. 

function processUserInput(name, callback) {
  const formattedName = name.trim().toUpperCase();
  callback(formattedName);
}

function displayResult(value) {
  console.log("Result:", value);
}

processUserInput("  nobu  ", displayResult);
// Result: NOBU

What is happening?

  1. displayResult is passed into processUserInput.
  2. processUserInput cleans up the input.
  3. Then it calls the callback with the cleaned value.

A callback lets one function say: “I will do the main work. You give me the custom behavior.”

3. What is a Higher-Order Function?

A higher-order function is a function that does at least one of these things:

  • Accepts another function as an argument
  • Returns another function

Example: accepts a function

function repeat(times, action) {
  for (let i = 0; i < times; i++) {
    action(i);
  }
}

repeat(3, function(index) {
  console.log("Running", index);
});

Example: returns a function

function createMultiplier(multiplier) {
  return function(number) {
    return number * multiplier;
  };
}

const double = createMultiplier(2);
const triple = createMultiplier(3);

console.log(double(10)); // 20
console.log(triple(10)); // 30

This second example also uses a closure. The returned function remembers the value of multiplier even after createMultiplier has finished running.

4. Array Methods Use Callbacks

Many everyday JavaScript methods are higher-order functions because they accept callback functions.

Method Purpose Callback Meaning
map Transforms every item How should each item be changed?
filter Keeps selected items Should this item stay?
reduce Combines values into one result How should the running result be updated?
forEach Runs side effects What should happen for each item?

map

const prices = [10, 20, 30];

const pricesWithTax = prices.map(function(price) {
  return price * 1.08;
});

console.log(pricesWithTax); // [10.8, 21.6, 32.4]

filter

const users = [
  { name: "Aki", active: true },
  { name: "Ben", active: false },
  { name: "Cara", active: true }
];

const activeUsers = users.filter(function(user) {
  return user.active;
});

console.log(activeUsers);

reduce

const cart = [
  { item: "Keyboard", price: 80 },
  { item: "Mouse", price: 30 },
  { item: "Monitor", price: 220 }
];

const total = cart.reduce(function(sum, product) {
  return sum + product.price;
}, 0);

console.log(total); // 330

5. Callbacks in Browser Events

Event listeners are one of the most common places where callbacks appear in web development. 

const button = document.querySelector("#saveButton");

button.addEventListener("click", function(event) {
  console.log("Button clicked");
  console.log(event.target);
});

The function passed to addEventListener is a callback. The browser calls it later when the user clicks the button.

Important idea

You do not call the callback yourself. You give it to the browser, and the browser calls it when the event happens. 

// Correct
button.addEventListener("click", handleClick);

// Incorrect if you want it to wait for the click
button.addEventListener("click", handleClick());

handleClick passes the function. handleClick() calls the function immediately.

6. Callbacks and Asynchronous Code

JavaScript uses callbacks heavily for asynchronous work: timers, file operations, network requests, and user events. 

console.log("Start");

setTimeout(function() {
  console.log("Timer finished");
}, 1000);

console.log("End");

Output:

Start
End
Timer finished

The callback passed to setTimeout runs later. JavaScript does not stop and wait for it. This is closely related to the event loop.

Modern replacement: Promises and async/await

Older JavaScript often used deeply nested callbacks. Modern JavaScript usually uses Promises and async/await, but callbacks are still everywhere underneath. 

// Callback style
loadUser(function(user) {
  loadOrders(user.id, function(orders) {
    console.log(orders);
  });
});

// Promise / async style
const user = await loadUser();
const orders = await loadOrders(user.id);
console.log(orders);

7. How This Applies to Web App Development

Forms

Callbacks validate input, submit forms, display errors, and update the UI.

form.addEventListener("submit", handleSubmit);

Search Boxes

Callbacks run when users type. They are often combined with debounce.

searchInput.addEventListener("input", handleSearch);

React Props

React components often receive callback functions from parent components.

<SaveButton onSave={saveUser} />

Data Processing

Higher-order array methods transform API data before rendering it.

users.filter(isActive).map(toUserCard);

Realistic Web App Example

const products = [
  { name: "Laptop", price: 1200, inStock: true },
  { name: "Mouse", price: 30, inStock: true },
  { name: "Monitor", price: 250, inStock: false }
];

function isAvailable(product) {
  return product.inStock;
}

function toListItem(product) {
  return `<li>${product.name} - $${product.price}</li>`;
}

const html = products
  .filter(isAvailable)
  .map(toListItem)
  .join("");

document.querySelector("#productList").innerHTML = html;

In this example, filter and map are higher-order functions. isAvailable and toListItem are callbacks.

8. Common Mistakes

Mistake 1: Calling the function too early

// Wrong
button.addEventListener("click", saveUser());

// Right
button.addEventListener("click", saveUser);

Mistake 2: Making callbacks too large

// Hard to read
users.filter(function(user) {
  return user.active && user.role === "admin" && user.lastLoginDays < 30;
});

// Easier to read
function isRecentActiveAdmin(user) {
  return user.active && user.role === "admin" && user.lastLoginDays < 30;
}

users.filter(isRecentActiveAdmin);

Mistake 3: Ignoring return values

const numbers = [1, 2, 3];

// Wrong: map expects a returned value
const doubled = numbers.map(function(n) {
  n * 2;
});

console.log(doubled); // [undefined, undefined, undefined]

// Right
const fixed = numbers.map(function(n) {
  return n * 2;
});

9. Best Practices

  • Use named callback functions when the logic is important or reused.
  • Use short arrow functions for simple transformations.
  • Keep callbacks small and focused.
  • Prefer map, filter, and reduce for data transformations.
  • Use forEach when you intentionally want side effects.
  • Do not call a callback when passing it to an event listener.
  • Use Promises or async/await instead of deeply nested asynchronous callbacks.
