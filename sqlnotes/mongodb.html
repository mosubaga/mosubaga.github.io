MongoDB Query Cheatsheet

Overview

A quick reference guide for querying documents in MongoDB.

Basic Query Operations

Find Documents

Find all documents in a collection: 

db.collection.find()

Find documents matching a condition: 

db.collection.find({ status: "active" })

Find one document:

db.collection.findOne({ _id: ObjectId("...") })

Adding Documents

MongoDB stores records as documents. Use insertOne() to add one document or insertMany() to add several documents at once.

Insert One Document

// Add one user document
db.users.insertOne({
  name: "Avery Chen",
  email: "avery@example.com",
  role: "editor",
  active: true,
  createdAt: new Date()
})

MongoDB creates a unique _id automatically when one is not provided.

Insert Multiple Documents

// Add several users in one operation
db.users.insertMany([
  {
    name: "Jordan Lee",
    email: "jordan@example.com",
    role: "viewer",
    active: true
  },
  {
    name: "Morgan Diaz",
    email: "morgan@example.com",
    role: "editor",
    active: true
  }
])

Modifying Documents

Use update operators such as $set, $inc, and $unset to change selected fields without replacing the entire document.

Update One Document

// Change selected fields on the first matching document
db.users.updateOne(
  { email: "avery@example.com" },
  {
    $set: {
      role: "admin",
      updatedAt: new Date()
    }
  }
)

Update Multiple Documents

// Increment loginCount for every active editor
db.users.updateMany(
  { role: "editor", active: true },
  {
    $inc: { loginCount: 1 },
    $set: { updatedAt: new Date() }
  }
)

Tip: Verify the filter with find() before running a broad update.

Remove a Field or Create When Missing 

// Remove a field from one document
db.users.updateOne(
  { email: "avery@example.com" },
  { $unset: { temporaryNote: "" } }
)

// Update a document, or insert it when no match exists
db.users.updateOne(
  { email: "sam@example.com" },
  { $set: { name: "Sam Rivera", active: true } },
  { upsert: true }
)

Deleting Documents

Use deleteOne() to remove the first matching document or deleteMany() to remove every document that matches a filter.

Delete One Document

// Delete one user by its unique ID
db.users.deleteOne({
  _id: ObjectId("64f1234567890abcdef1234")
})

Delete Multiple Documents

// Delete all inactive users last seen before 2024
db.users.deleteMany({
  active: false,
  lastSeenAt: { $lt: ISODate("2024-01-01T00:00:00Z") }
})

⚠️ Delete Carefully

An empty filter, deleteMany({}), deletes every document in the collection. Run the same filter with find() or countDocuments() first, and make sure you have a current backup.

Query Operators

Comparison Operators

// Equal to
db.collection.find({ age: 25 })

// Greater than
db.collection.find({ age: { $gt: 25 } })

// Greater than or equal
db.collection.find({ age: { $gte: 25 } })

// Less than
db.collection.find({ age: { $lt: 25 } })

// Less than or equal
db.collection.find({ age: { $lte: 25 } })

// Not equal
db.collection.find({ status: { $ne: "inactive" } })

// In array
db.collection.find({ status: { $in: ["active", "pending"] } })

// Not in array
db.collection.find({ status: { $nin: ["deleted", "archived"] } })

Logical Operators

// AND (implicit)
db.collection.find({ status: "active", age: { $gte: 18 } })

// AND (explicit)
db.collection.find({ $and: [
  { status: "active" },
  { age: { $gte: 18 } }
]})

// OR
db.collection.find({ $or: [
  { status: "active" },
  { priority: "high" }
]})

// NOT
db.collection.find({ age: { $not: { $gte: 18 } } })

// NOR
db.collection.find({ $nor: [
  { status: "deleted" },
  { archived: true }
]})

Field Projection

Include Specific Fields

db.collection.find({ status: "active" }, { name: 1, email: 1 })

Exclude Specific Fields

db.collection.find({}, { password: 0, internal_id: 0 })

You cannot mix inclusion and exclusion, except for the _id field.

Querying Arrays

Array Matching

Match exact array:

db.collection.find({ tags: ["mongodb", "database"] })

Match array containing element: 

db.collection.find({ tags: "mongodb" })

Match with $all: 

db.collection.find({ tags: { $all: ["mongodb", "nosql"] } })

Match by array size:

db.collection.find({ tags: { $size: 3 } })

Array Element Queries

Query array elements:

db.collection.find({ "tags.0": "mongodb" })

$elemMatch for complex conditions: 

db.collection.find({
  scores: { $elemMatch: { $gte: 80, $lt: 90 } }
})

Querying Nested Documents

Dot Notation

db.collection.find({ "address.city": "New York" })

Match Entire Nested Document

db.collection.find({
  address: { street: "123 Main St", city: "NYC" }
})

Regular Expressions

Regex Query Examples

// Case-sensitive match
db.collection.find({ name: /^John/ })

// Case-insensitive match
db.collection.find({ name: { $regex: /john/i } })

// Using $options
db.collection.find({ name: { $regex: "^john", $options: "i" } })

Existence and Type Checks

Check if Field Exists

db.collection.find({ email: { $exists: true } })

Check Field Type

db.collection.find({ age: { $type: "number" } })
db.collection.find({ age: { $type: 16 } }) // 16 = 32-bit integer

Sorting, Limiting & Skipping

Sort and Page Results

Sort ascending:

db.collection.find().sort({ age: 1 })

Sort descending:

db.collection.find().sort({ createdAt: -1 })

Limit results:

db.collection.find().limit(10)

Skip results for pagination:

db.collection.find().skip(20).limit(10)

Combine operations:

db.collection.find({ status: "active" })
  .sort({ createdAt: -1 })
  .skip(10)
  .limit(5)

Counting and Distinct Values

Counting Documents

Count all documents:

db.collection.countDocuments()

Count with filter:

db.collection.countDocuments({ status: "active" })

Estimated count, faster but less accurate: 

db.collection.estimatedDocumentCount()

Distinct Values

db.collection.distinct("category")
db.collection.distinct("city", { status: "active" })

Text Search

Create Text Index

db.collection.createIndex({ content: "text" })

Perform Text Search

db.collection.find({ $text: { $search: "mongodb query" } })

Text Search with Score

db.collection.find(
  { $text: { $search: "database" } },
  { score: { $meta: "textScore" } }
).sort({ score: { $meta: "textScore" } })

Aggregation Pipeline Basics

Pipeline Example

db.collection.aggregate([
  // Match stage (filter)
  { $match: { status: "active" } },

  // Group stage
  { $group: {
    _id: "$category",
    total: { $sum: 1 },
    avgPrice: { $avg: "$price" }
  }},

  // Sort stage
  { $sort: { total: -1 } },

  // Limit stage
  { $limit: 5 }
])

Common Query Patterns

Date Queries

// Documents from today
db.collection.find({
  createdAt: {
    $gte: new Date(new Date().setHours(0,0,0,0))
  }
})

// Documents from last 7 days
db.collection.find({
  createdAt: {
    $gte: new Date(Date.now() - 7*24*60*60*1000)
  }
})

// Date range
db.collection.find({
  createdAt: {
    $gte: ISODate("2025-01-01"),
    $lt: ISODate("2025-12-31")
  }
})

NULL and Missing Values

// Field is null
db.collection.find({ field: null })

// Field is null OR does not exist
db.collection.find({ field: { $in: [null] } })

// Field exists but is null
db.collection.find({ field: { $type: "null" } })

// Field does not exist
db.collection.find({ field: { $exists: false } })
