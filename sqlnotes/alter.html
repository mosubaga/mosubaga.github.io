ALTER TABLE Command

The ALTER TABLE statement is used to modify an existing table structure in SQL. You can add, delete, or modify columns, change data types, add constraints, and more.

Basic Syntax:

ALTER TABLE table_name
ACTION column_name datatype;

Adding a Column

Use the ADD keyword to add a new column to an existing table.

Example: Add a Single Column

-- Add an email column to the employees table
ALTER TABLE employees
ADD email VARCHAR(100);

Example: Add Multiple Columns

-- Add multiple columns at once
ALTER TABLE employees
ADD phone VARCHAR(20),
ADD address VARCHAR(200),
ADD hire_date DATE;

Example: Add Column with Default Value

-- Add a column with a default value
ALTER TABLE employees
ADD status VARCHAR(20) DEFAULT 'active';

Dropping (Removing) a Column

Use the DROP COLUMN keyword to remove a column from a table. Be careful—this permanently deletes the column and all its data!

Example: Drop a Single Column

-- Remove the middle_name column
ALTER TABLE employees
DROP COLUMN middle_name;

Example: Drop Multiple Columns

-- Remove multiple columns
ALTER TABLE employees
DROP COLUMN fax,
DROP COLUMN pager;

⚠️ Warning

Dropping a column is irreversible. Always backup your data before performing destructive operations. Some databases may not allow dropping columns that are referenced by foreign keys or indexes.

Modifying Column Data Types

You can change the data type of an existing column. The syntax varies slightly between different database systems (MySQL uses MODIFY, PostgreSQL and SQL Server use ALTER COLUMN).

MySQL Syntax

-- Change the data type of a column in MySQL
ALTER TABLE employees
MODIFY COLUMN phone VARCHAR(25);

-- Change with NOT NULL constraint
ALTER TABLE employees
MODIFY COLUMN salary DECIMAL(12, 2) NOT NULL;

PostgreSQL Syntax

-- Change the data type in PostgreSQL
ALTER TABLE employees
ALTER COLUMN phone TYPE VARCHAR(25);

-- With USING clause for data conversion
ALTER TABLE employees
ALTER COLUMN age TYPE INTEGER USING age::integer;

SQL Server Syntax

-- Change the data type in SQL Server
ALTER TABLE employees
ALTER COLUMN phone VARCHAR(25);

Renaming Columns

Rename existing columns to better reflect their purpose or follow naming conventions.

MySQL Syntax

-- Rename a column in MySQL
ALTER TABLE employees
CHANGE COLUMN old_name new_name VARCHAR(100);

-- Or using RENAME COLUMN (MySQL 8.0+)
ALTER TABLE employees
RENAME COLUMN old_name TO new_name;

PostgreSQL / SQL Server Syntax

-- Rename a column in PostgreSQL
ALTER TABLE employees
RENAME COLUMN old_name TO new_name;

-- SQL Server uses sp_rename
EXEC sp_rename 'employees.old_name', 'new_name', 'COLUMN';

Adding and Dropping Constraints

You can add or remove constraints like PRIMARY KEY, FOREIGN KEY, UNIQUE, CHECK, and NOT NULL.

Add Constraints

-- Add a primary key
ALTER TABLE employees
ADD PRIMARY KEY (employee_id);

-- Add a foreign key
ALTER TABLE orders
ADD FOREIGN KEY (customer_id) REFERENCES customers(id);

-- Add a unique constraint
ALTER TABLE employees
ADD UNIQUE (email);

-- Add a check constraint
ALTER TABLE employees
ADD CHECK (salary > 0);

-- Add NOT NULL constraint (varies by database)
ALTER TABLE employees
MODIFY COLUMN email VARCHAR(100) NOT NULL;  -- MySQL

ALTER TABLE employees
ALTER COLUMN email SET NOT NULL;  -- PostgreSQL

Drop Constraints

-- Drop a primary key
ALTER TABLE employees
DROP PRIMARY KEY;

-- Drop a foreign key (need constraint name)
ALTER TABLE orders
DROP FOREIGN KEY fk_customer_id;

-- Drop a check constraint
ALTER TABLE employees
DROP CHECK check_salary;

-- Remove NOT NULL constraint
ALTER TABLE employees
MODIFY COLUMN email VARCHAR(100) NULL;  -- MySQL

ALTER TABLE employees
ALTER COLUMN email DROP NOT NULL;  -- PostgreSQL

Renaming Tables

You can rename an entire table using ALTER TABLE or RENAME TABLE.

Example

-- MySQL
RENAME TABLE old_table_name TO new_table_name;

-- Or using ALTER TABLE
ALTER TABLE old_table_name
RENAME TO new_table_name;

-- PostgreSQL
ALTER TABLE old_table_name
RENAME TO new_table_name;

-- SQL Server
EXEC sp_rename 'old_table_name', 'new_table_name';

Practical Example: Complete Table Modification

Let's see a real-world scenario where we modify an existing employees table:

Original Table Structure

CREATE TABLE employees (
    id INT,
    name VARCHAR(50),
    dept VARCHAR(20),
    pay DECIMAL(8, 2)
);

Modifications

-- Add primary key
ALTER TABLE employees
ADD PRIMARY KEY (id);

-- Rename columns for clarity
ALTER TABLE employees
RENAME COLUMN name TO full_name;

ALTER TABLE employees
RENAME COLUMN dept TO department;

ALTER TABLE employees
RENAME COLUMN pay TO salary;

-- Change data types
ALTER TABLE employees
MODIFY COLUMN full_name VARCHAR(100);

ALTER TABLE employees
MODIFY COLUMN salary DECIMAL(12, 2) NOT NULL;

-- Add new columns
ALTER TABLE employees
ADD email VARCHAR(100) UNIQUE,
ADD hire_date DATE,
ADD status VARCHAR(20) DEFAULT 'active';

-- Add check constraint
ALTER TABLE employees
ADD CHECK (salary >= 0);

Final Table Structure

--     full_name VARCHAR(100),
--     department VARCHAR(20),
--     salary DECIMAL(12, 2) NOT NULL CHECK(salary >= 0),
--     email VARCHAR(100) UNIQUE,
--     hire_date DATE,
--     status VARCHAR(20) DEFAULT 'active'
-- )

Best Practices

  • Always backup your database before making structural changes
  • Test ALTER statements in a development environment first
  • Be aware that some operations may lock the table temporarily
  • Check for dependencies (foreign keys, views, stored procedures) before dropping columns
  • When changing data types, ensure existing data is compatible with the new type
  • Use transactions when making multiple changes to allow rollback if needed
  • Document all schema changes for team members and future reference
