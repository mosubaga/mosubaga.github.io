SQL User Management

Overview

User management in SQL involves creating users, granting them appropriate permissions, and controlling their access to databases and tables. Proper user management is crucial for database security and access control.

Creating Users

To create a new database user, you need administrative privileges. The syntax varies slightly between different SQL databases (MySQL, PostgreSQL, SQL Server, etc.).

MySQL - Create User

-- Create a new user with password
CREATE USER 'username'@'localhost' IDENTIFIED BY 'password123';

-- Create a user that can connect from any host
CREATE USER 'username'@'%' IDENTIFIED BY 'password123';

PostgreSQL - Create User

-- Create a new user (role) with password
CREATE USER username WITH PASSWORD 'password123';

-- Create a user with login privilege
CREATE ROLE username WITH LOGIN PASSWORD 'password123';

SQL Server - Create User

-- First create a login
CREATE LOGIN username WITH PASSWORD = 'password123';

-- Then create a user for a specific database
USE my_database;
CREATE USER username FOR LOGIN username;

Granting Permissions

After creating a user, you need to grant them specific permissions to perform actions on databases and tables. Use the GRANT statement to assign privileges.

Grant SELECT Permission

Allow a user to read data from a table:

-- Grant SELECT on a specific table
GRANT SELECT ON database_name.table_name TO 'username'@'localhost';

-- Grant SELECT on all tables in a database
GRANT SELECT ON database_name.* TO 'username'@'localhost';

Grant INSERT, UPDATE, DELETE Permissions

Allow a user to modify data:

-- Grant multiple permissions on a table
GRANT INSERT, UPDATE, DELETE ON database_name.table_name 
TO 'username'@'localhost';

-- Grant specific permissions on multiple tables
GRANT INSERT, UPDATE ON database_name.* TO 'username'@'localhost';

Grant CREATE and DROP Permissions

Allow a user to create and delete database objects:

-- Grant CREATE permission
GRANT CREATE ON database_name.* TO 'username'@'localhost';

-- Grant DROP permission
GRANT DROP ON database_name.* TO 'username'@'localhost';

-- Grant both CREATE and DROP
GRANT CREATE, DROP ON database_name.* TO 'username'@'localhost';

Grant EXECUTE Permission

Allow a user to execute stored procedures and functions:

-- Grant EXECUTE on a specific procedure
GRANT EXECUTE ON PROCEDURE database_name.procedure_name 
TO 'username'@'localhost';

-- Grant EXECUTE on all procedures in a database
GRANT EXECUTE ON database_name.* TO 'username'@'localhost';

Granting All Privileges

You can grant all available privileges to a user on a database. This gives the user full control over the specified database, similar to administrative rights.

Grant All Privileges on a Database

-- MySQL: Grant all privileges on a specific database
GRANT ALL PRIVILEGES ON database_name.* TO 'username'@'localhost';

-- Apply the changes
FLUSH PRIVILEGES;

Grant All Privileges on All Databases

⚠️ Use with caution - this gives full system access:

-- Grant all privileges on all databases
GRANT ALL PRIVILEGES ON *.* TO 'username'@'localhost';

-- With grant option (allows user to grant privileges to others)
GRANT ALL PRIVILEGES ON *.* TO 'username'@'localhost' WITH GRANT OPTION;

-- Apply the changes
FLUSH PRIVILEGES;

PostgreSQL - Grant All Privileges

-- Grant all privileges on a database
GRANT ALL PRIVILEGES ON DATABASE database_name TO username;

-- Grant all on all tables in a schema
GRANT ALL PRIVILEGES ON ALL TABLES IN SCHEMA public TO username;

-- Grant all on future tables (for newly created tables)
ALTER DEFAULT PRIVILEGES IN SCHEMA public 
GRANT ALL ON TABLES TO username;

⚠️ Security Best Practices

  • Only grant the minimum privileges necessary (Principle of Least Privilege)
  • Avoid using ALL PRIVILEGES unless absolutely necessary
  • Use specific permissions for specific tasks
  • Regularly review and audit user permissions
  • Remove privileges when they are no longer needed
  • Use strong passwords for database users

Viewing and Revoking Permissions

View User Permissions

-- MySQL: Show grants for a user
SHOW GRANTS FOR 'username'@'localhost';

-- Show grants for current user
SHOW GRANTS;

Revoke Permissions

-- Revoke specific permission
REVOKE SELECT ON database_name.* FROM 'username'@'localhost';

-- Revoke all privileges
REVOKE ALL PRIVILEGES ON database_name.* FROM 'username'@'localhost';

-- Apply the changes
FLUSH PRIVILEGES;

Delete a User

-- MySQL: Drop a user
DROP USER 'username'@'localhost';

-- PostgreSQL: Drop a user
DROP USER username;
