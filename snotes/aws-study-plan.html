Practical tips
Use IAM roles, not keys
Never hardcode AWS credentials in code. Assign IAM roles to Lambda and EC2 instances. Store secrets in SSM Parameter Store or Secrets Manager.
Stop EC2 instances when idle
The free tier gives 750 hours/month — enough for one t2.micro running 24/7. If you spin up multiple instances, stop them when not in use.
Avoid NAT Gateways
NAT Gateways cost ~$0.045/hour and are NOT free tier. For learning purposes, deploy resources in public subnets or use VPC endpoints instead.
Pair with AWS Skill Builder
AWS Skill Builder has free digital courses mapped to each service. The Cloud Practitioner Essentials course is a great companion to Phases 1–3.
Tag every resource
Add a tag like Project=learning to everything you create. Makes cleanup — and billing analysis — much easier.
Delete resources after each phase
Before moving to the next phase, delete resources from the previous one (RDS snapshots, EC2 AMIs, etc.). Use CloudFormation stacks so teardown is one command.
