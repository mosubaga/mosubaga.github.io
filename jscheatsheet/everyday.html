Everyday Patterns

These are the habits that make TypeScript and JavaScript feel smooth: safer property access, reliable defaults, clean object/array unpacking, and flexible argument handling. Master these four patterns and your code gets shorter, clearer, and more defensive without extra noise.

Optional Chaining (?.)

Read deeply nested values without exploding on null or undefined.


const user = { profile: { name: 'Aiko' } };

const name = user.profile?.name; // 'Aiko'
const city = user.address?.city; // undefined

// Also works with method calls
const label = user.profile?.formatName?.() ?? 'Anonymous';

Nullish Coalescing (??)

Provide a default only when the value is null or undefined (not for 0, '', or false).


const scoops = 0;
const shownScoops = scoops ?? 2; // 0 (keeps intentional zero)

const flavor = undefined;
const chosenFlavor = flavor ?? 'Vanilla'; // 'Vanilla'

Destructuring

Pull values out of arrays and objects with clear, compact syntax.


const colors = ['Strawberry', 'Matcha', 'Mocha'];
const [first, , third] = colors; // 'Strawberry', 'Mocha'

const order = { flavor: 'Mango', size: 'Large', topping: 'Boba' };
const { flavor, topping } = order;

const { size: cupSize = 'Medium' } = order; // rename + default

Spread & Rest (...)

Spread expands values; rest gathers them. Same dots, different intent.


// Spread: copy/merge
const base = { flavor: 'Vanilla', scoops: 2 };
const deluxe = { ...base, topping: 'Sprinkles' };

const list = [1, 2, 3];
const extended = [...list, 4, 5];

// Rest: collect remaining values
function buildSundae(flavor: string, ...toppings: string[]) {
  return { flavor, toppings };
}

const [head, ...tail] = extended; // head = 1, tail = [2,3,4,5]
