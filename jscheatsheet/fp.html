Functional Programming in JavaScript

Functional programming in JavaScript is less about academic purity and more about writing code that is easier to read, test, and change. The main idea is to build programs from small functions that transform data instead of mutating shared state. In practice, that means preferring pure functions, immutable updates, array methods like map and filter, and composition patterns that let simple steps snap together cleanly.

Pure Functions

A pure function returns the same output for the same input and does not change anything outside itself.


function addTax(price, taxRate) {
  return price + price * taxRate;
}

const total = addTax(20, 0.08); // 21.6

Immutable Updates

Instead of changing arrays or objects in place, create new ones with the updated values.


const cart = [
  { name: 'Mochi', price: 4 },
  { name: 'Taiyaki', price: 6 }
];

const updatedCart = [...cart, { name: 'Matcha Cookie', price: 3 }];

const order = { status: 'pending', total: 10 };
const completedOrder = { ...order, status: 'complete' };

Transform Data with map, filter, reduce

These array methods are core tools for functional-style JavaScript.


const numbers = [1, 2, 3, 4, 5];

const doubled = numbers.map((n) => n * 2);
const evens = numbers.filter((n) => n % 2 === 0);
const sum = numbers.reduce((total, n) => total + n, 0);

Function Composition

Break a task into small functions and pipe values through them one step at a time.


const trim = (text) => text.trim();
const toLower = (text) => text.toLowerCase();
const slugify = (text) => text.replaceAll(' ', '-');

const compose = (...fns) => (value) =>
  fns.reduceRight((current, fn) => fn(current), value);

const makeSlug = compose(slugify, toLower, trim);

makeSlug('  Functional JavaScript Rocks  ');
// 'functional-javascript-rocks'

Higher-Order Functions

Functions can receive other functions or return them, which makes logic reusable and configurable.


function withLogger(fn) {
  return function (...args) {
    console.log('Calling with', args);
    return fn(...args);
  };
}

const multiply = (a, b) => a * b;
const loggedMultiply = withLogger(multiply);

loggedMultiply(3, 4); // 12

Practical FP Rules

Use functional programming where it improves clarity, not as a rigid rulebook.


// Good habits:
// 1. Prefer const by default
// 2. Return new values instead of mutating inputs
// 3. Keep functions focused on one job
// 4. Push side effects to the edges of the app

function checkout(items, discountRate) {
  const subtotal = items.reduce((sum, item) => sum + item.price, 0);
  const discount = subtotal * discountRate;
  return subtotal - discount;
}
