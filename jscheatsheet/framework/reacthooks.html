Before choosing a hook

Rules and Mental Models

Call hooks at the top level

Do not call hooks inside loops, conditions, nested functions, or event handlers. React relies on the same call order on every render.

Use dependencies honestly

Every value read inside an effect, memo, or callback should appear in its dependency array unless it is stable by design.

Prefer state only for rendering

If changing a value should update the UI, use state. If it should persist without causing a render, use a ref.

Memoize after measuring

useMemo and useCallback help when recomputation or child rerenders are actually expensive.

1

useState

Stores local component state and returns the current value plus an updater function.

  • Best for form fields, toggles, counters, selected tabs, and local UI state.
  • Use functional updates when the next state depends on the previous state.
  • Keep related state together when it changes together.
const [count, setCount] = useState(0);

function increment() {
  setCount(previous => previous + 1);
}
2

useEffect

Runs side effects after React paints the screen, such as fetching data or subscribing to external systems.

  • No dependency array: runs after every render.
  • Empty array: runs once after mount, then cleanup on unmount.
  • Dependency array: reruns when listed values change.
useEffect(() => {
  const controller = new AbortController();

  fetch('/api/users', { signal: controller.signal });

  return () => controller.abort();
}, []);
3

useContext

Reads a value from the nearest matching context provider without passing props through every level.

  • Good for theme, locale, authenticated user, feature flags, and app-level services.
  • Changing a provider value rerenders consumers below it.
  • Split contexts when unrelated values update at different speeds.
const ThemeContext = createContext('light');

function Toolbar() {
  const theme = useContext(ThemeContext);
  return <button className={theme}>Save</button>;
}
4

useReducer

Manages state transitions with a reducer function, which is useful when updates become branching or event-driven.

  • Good for multi-field forms, workflows, undoable state, and related state transitions.
  • Actions describe what happened; the reducer decides the next state.
  • Reducers should stay pure and return new objects instead of mutating state.
function reducer(state, action) {
  switch (action.type) {
    case 'open':
      return { ...state, isOpen: true };
    default:
      return state;
  }
}

const [state, dispatch] = useReducer(reducer, { isOpen: false });
5

useRef

Stores a mutable value whose .current property survives renders without triggering another render.

  • Use it for DOM nodes, timer IDs, previous values, and integration handles.
  • Do not read or write refs during render when the value affects output.
  • For visible UI changes, use useState instead.
const inputRef = useRef(null);

function focusSearch() {
  inputRef.current?.focus();
}

return <input ref={inputRef} />;
6

useMemo

Caches the result of an expensive calculation until one of its dependencies changes.

  • Useful for filtering, sorting, formatting, or deriving large data sets.
  • It is a performance hint, not a semantic guarantee.
  • Avoid wrapping every calculation by default.
const visibleTodos = useMemo(() => {
  return todos.filter(todo => todo.status === filter);
}, [todos, filter]);
7

useCallback

Returns a stable function reference until its dependencies change.

  • Helpful when passing callbacks to memoized children.
  • Useful when a function is itself a dependency of another hook.
  • It does not prevent the function body from running when called.
const handleSave = useCallback(() => {
  saveDraft(draftId, title);
}, [draftId, title]);

return <Editor onSave={handleSave} />;
8

useLayoutEffect

Runs synchronously after DOM mutations but before the browser paints.

  • Use it for layout reads, measurements, and positioning that must avoid flicker.
  • Prefer useEffect unless the timing matters visually.
  • Keep the work small because it blocks painting.
useLayoutEffect(() => {
  const box = panelRef.current.getBoundingClientRect();
  setPanelHeight(box.height);
}, [items.length]);
9

useImperativeHandle

Customizes the value exposed to a parent component through a ref.

  • Usually paired with forwardRef.
  • Good for exposing focused methods like focus, reset, or scrollToTop.
  • Avoid exposing broad internal component state.
useImperativeHandle(ref, () => ({
  focus() {
    inputRef.current?.focus();
  }
}), []);
10

useDebugValue

Displays a label for custom hooks in React DevTools.

  • Most useful inside reusable custom hooks.
  • Keep labels short and focused on the hook state.
  • Use lazy formatting if creating the label is expensive.
function useOnlineStatus() {
  const isOnline = useNetworkStore();

  useDebugValue(isOnline ? 'Online' : 'Offline');
  return isOnline;
}

Fast decisions

Which Hook Should I Reach For?

Need Hook Reason
UI changes when a value changes useState or useReducer State updates schedule renders.
Talk to the outside world useEffect Effects synchronize React with APIs, subscriptions, timers, and the DOM.
Share app-wide values useContext Consumers read values from a provider without prop drilling.
Persist a value without rendering useRef Refs survive renders while staying mutable.
Avoid costly recalculation useMemo The computed result is reused while dependencies stay the same.
Pass a stable function useCallback The function identity remains stable between dependency changes.
