Install

Use Redux Toolkit for Redux logic and React-Redux for connecting React components to the store.

  • Redux Toolkit is the default modern Redux API.
  • React-Redux provides Provider, useSelector, and useDispatch.
  • The Redux DevTools browser extension helps inspect actions and state changes.
# npm
npm install @reduxjs/toolkit react-redux

# pnpm
pnpm add @reduxjs/toolkit react-redux
2

Store

The store holds the current application state and runs reducers whenever actions are dispatched.

  • Create it once for the app.
  • Pass slice reducers into configureStore.
  • Redux Toolkit sets up useful defaults like DevTools and middleware.
import { configureStore } from '@reduxjs/toolkit';
import todosReducer from './features/todos/todosSlice';

export const store = configureStore({
  reducer: {
    todos: todosReducer
  }
});
3

Provider

Provider makes the store available to React components below it.

  • Wrap your app near the root.
  • Pass the same store instance for the lifetime of the app.
  • Components can then use React-Redux hooks.
import { Provider } from 'react-redux';
import { store } from './store';

root.render(
  <Provider store={store}>
    <App />
  </Provider>
);
4

Slice

A slice groups reducer logic, action creators, and initial state for one feature.

  • Write reducers as if mutating state; Immer safely produces immutable updates.
  • Each reducer becomes an action creator.
  • Use feature-based files like features/cart/cartSlice.js.
const todosSlice = createSlice({
  name: 'todos',
  initialState: [],
  reducers: {
    todoAdded(state, action) {
      state.push({
        id: nanoid(),
        text: action.payload,
        completed: false
      });
    }
  }
});
5

Actions

Actions are events. They describe what happened, while reducers decide how state changes.

  • Action objects have a type field.
  • Extra data usually goes in payload.
  • Dispatch action creators generated by slices.
export const { todoAdded, todoToggled } = todosSlice.actions;

dispatch(todoAdded('Learn Redux Toolkit'));

// Resulting action shape
{
  type: 'todos/todoAdded',
  payload: 'Learn Redux Toolkit'
}
6

useSelector

useSelector reads data from the Redux store and rerenders when the selected value changes.

  • Select the smallest data the component needs.
  • Avoid returning new objects or arrays unless they are memoized.
  • Move repeated lookup logic into selector functions.
const selectTodos = state => state.todos;
const selectCompleted = state =>
  state.todos.filter(todo => todo.completed);

function TodoList() {
  const todos = useSelector(selectTodos);
  return todos.map(todo => <Todo key={todo.id} todo={todo} />);
}
7

useDispatch

useDispatch returns the store dispatch function so components can send actions.

  • Dispatch from event handlers, effects, or callbacks.
  • Keep business logic in slices, thunks, or services when it grows.
  • In TypeScript, prefer typed hooks like useAppDispatch.
function AddTodo() {
  const dispatch = useDispatch();

  function handleSubmit(text) {
    dispatch(todoAdded(text));
  }
}
8

Async Thunks

createAsyncThunk handles promise lifecycles with pending, fulfilled, and rejected actions.

  • Use thunks for request logic, conditional async work, or orchestration.
  • Track loading and error state in extraReducers.
  • Use RTK Query for most server-cache CRUD workflows.
export const fetchTodos = createAsyncThunk(
  'todos/fetchTodos',
  async () => {
    const response = await fetch('/api/todos');
    return response.json();
  }
);
9

extraReducers

Use extraReducers when a slice reacts to actions it did not define.

  • Most common with async thunk lifecycle actions.
  • The builder callback keeps action matching type-safe and readable.
  • Reducers still must stay synchronous.
extraReducers: builder => {
  builder
    .addCase(fetchTodos.pending, state => {
      state.status = 'loading';
    })
    .addCase(fetchTodos.fulfilled, (state, action) => {
      state.status = 'succeeded';
      state.items = action.payload;
    });
}
10

RTK Query

RTK Query is Redux Toolkit's data fetching and caching layer.

  • Use it for API cache, loading states, deduping, invalidation, and refetching.
  • It generates React hooks from endpoint definitions.
  • Prefer it over hand-written thunks for typical server data.
const api = createApi({
  baseQuery: fetchBaseQuery({ baseUrl: '/api' }),
  endpoints: builder => ({
    getTodos: builder.query({
      query: () => 'todos'
    })
  })
});

export const { useGetTodosQuery } = api;

Fast decisions

When Should I Use Redux?

Situation Good choice Why
Local form input or one component toggle useState Keep simple state near the component that owns it.
Shared app state used across many screens createSlice Centralized updates make behavior easier to trace.
Complex update rules with many events Reducers Actions separate what happened from how state changes.
Derived values from store state Selectors Selectors keep component reads consistent and reusable.
Server data with caching needs RTK Query It handles loading, caching, invalidation, and refetching.

Common mistakes

Redux Gotchas

Putting everything in Redux

Use Redux for shared, important, or complex state. Keep purely local UI state inside components.

Mutating outside slices

Immer is active inside Toolkit reducers. Outside reducers, normal JavaScript mutation is still mutation.

Unstable selectors

Returning fresh arrays or objects from useSelector can rerender on every action.

Side effects in reducers

Reducers should be pure and synchronous. Put async work in thunks, listeners, RTK Query, or components.
