Objects Cheat Sheet

Objects in JavaScript are collections of key-value pairs that allow you to store and organize related data in a structured way. Unlike arrays, which use numeric indices, objects use named properties (keys) to access their values. You create objects using curly braces, and each property consists of a key and its associated value, separated by a colon. Objects are ideal for representing real-world entities with multiple attributes—for example, a user object might have properties like name, email, and age. They're fundamental to JavaScript programming and form the basis of more complex concepts like classes and prototypes, making them essential for organizing data and building applications.

Creation & Interfaces

// JavaScript
const candy = { name: 'Gummy Bear', flavor: 'Cherry' };

// TypeScript
interface Candy {
  name: string;
  flavor: string;
  rating?: number; // Optional property
}

const treat: Candy = { name: 'Lollipop', flavor: 'Strawberry' };

Accessing & Manipulating

Accessing properties

console.log(candy.name); // Dot notation
console.log(candy['flavor']); // Bracket notation

Adding/Updating

candy.color = 'Red';
candy.flavor = 'Sour Cherry';

Deleting

delete candy.color;

Object Methods

Object.keys(obj): Get all property names

const keys = Object.keys(candy); // ['name', 'flavor']

Object.values(obj): Get all property values

const values = Object.values(candy); // ['Gummy Bear', 'Cherry']

Object.entries(obj): Get key-value pairs

for (const [key, value] of Object.entries(candy)) {
  console.log(`${key}: ${value}`);
}

Object.assign(target, ...sources): Merge objects

const fullTreat = Object.assign({}, candy, { size: 'Large' });

Spread Operator

Shallow copy & merging

const copiedCandy = { ...candy };
const mergedCandy = { ...candy, brand: 'SweetCo' };

Computed & Shorthand Properties

Computed property names: use an expression inside brackets as a key

const field = 'color';
const candy = { [field]: 'Red', [`${field}Code`]: '#FF0000' };
// { color: 'Red', colorCode: '#FF0000' }

Shorthand properties: variable name becomes the key

const name = 'Gummy Bear';
const flavor = 'Cherry';

// Instead of { name: name, flavor: flavor }
const candy = { name, flavor };
// { name: 'Gummy Bear', flavor: 'Cherry' }

Method shorthand: define methods without function keyword

// Instead of { describe: function() { ... } }
const candy = {
  name: 'Lollipop',
  describe() {
    return `This is a ${this.name}`;
  }
};

Checking & Protecting

'key' in obj: Check if a property exists (including inherited)

const candy = { name: 'Gummy Bear', flavor: 'Cherry' };
console.log('name' in candy);    // true
console.log('rating' in candy);  // false

Object.hasOwn(obj, key): Modern way to check own properties (not inherited)

const candy = { name: 'Gummy Bear' };
console.log(Object.hasOwn(candy, 'name'));     // true
console.log(Object.hasOwn(candy, 'toString')); // false (inherited)

Object.freeze(obj): Prevent any modifications (shallow freeze)

const candy = { name: 'Gummy Bear', flavor: 'Cherry' };
Object.freeze(candy);
candy.flavor = 'Grape'; // Silently fails (throws in strict mode)
console.log(candy.flavor); // 'Cherry'

Object.keys(obj).length: Count the number of properties

const candy = { name: 'Gummy Bear', flavor: 'Cherry', color: 'Red' };
console.log(Object.keys(candy).length); // 3

Destructuring & Nesting

Nested destructuring: extract values from deeply nested objects

const candy = {
  name: 'Gummy Bear',
  details: { flavor: 'Cherry', color: 'Red' }
};
const { details: { flavor, color } } = candy;
console.log(flavor); // 'Cherry'
console.log(color);  // 'Red'

Default values: provide fallbacks for missing properties

const candy = { name: 'Gummy Bear' };
const { name, flavor = 'Unknown', rating = 0 } = candy;
console.log(flavor); // 'Unknown'
console.log(rating); // 0

Renaming: assign to a different variable name

const candy = { name: 'Gummy Bear', flavor: 'Cherry' };
const { name: candyName, flavor: candyFlavor } = candy;
console.log(candyName);   // 'Gummy Bear'
console.log(candyFlavor); // 'Cherry'

Destructuring function parameters: clean and readable signatures

function describeCandy({ name, flavor, rating = 'N/A' }) {
  return `${name} (${flavor}) - Rating: ${rating}`;
}
describeCandy({ name: 'Gummy Bear', flavor: 'Cherry' });
// 'Gummy Bear (Cherry) - Rating: N/A'

Map & Set

new Map(): Key-value pairs where keys can be any type (not just strings)

const inventory = new Map();
inventory.set('Gummy Bear', 50);
inventory.set('Lollipop', 30);

console.log(inventory.get('Gummy Bear')); // 50
console.log(inventory.has('Lollipop'));    // true
console.log(inventory.size);              // 2

inventory.delete('Lollipop');

// Iterate over entries
for (const [candy, count] of inventory) {
  console.log(`${candy}: ${count}`);
}

new Set(): A collection of unique values (duplicates are ignored)

const flavors = new Set();
flavors.add('Cherry');
flavors.add('Grape');
flavors.add('Cherry'); // Ignored, already exists

console.log(flavors.has('Cherry')); // true
console.log(flavors.size);          // 2

flavors.delete('Grape');

// Great for removing duplicates from an array
const arr = [1, 2, 2, 3, 3, 3];
const unique = [...new Set(arr)]; // [1, 2, 3]

When to use Map vs Object, Set vs Array

// Use Map when:
// - Keys are not strings (objects, numbers, etc.)
// - You need to know the size easily (.size)
// - You add/remove keys frequently

// Use Object when:
// - Keys are strings and known at compile time
// - You need JSON serialization
// - You want destructuring or spread syntax

// Use Set when:
// - You need unique values only
// - You check membership often (.has() is O(1))

// Use Array when:
// - Order matters and duplicates are allowed
// - You need map/filter/reduce operations
