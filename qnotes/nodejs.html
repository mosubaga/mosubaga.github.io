Node.js & Bun

JavaScript for the server-side, fast and efficient

What is Node.js?

Node.js is an open-source, cross-platform JavaScript runtime environment that executes JavaScript code outside a web browser. Created by Ryan Dahl in 2009, it revolutionized server-side development.

If you already know JavaScript from web development, Node.js lets you use those same skills for backend development, making it an excellent choice for full-stack developers.

What is Bun?

Bun is an all-in-one JavaScript runtime, bundler, transpiler, and package manager. It is designed to be a faster alternative to Node.js, with native TypeScript support.

Bun aims to be a drop-in replacement for Node.js while being significantly faster for many operations like installing packages and running scripts.

Key Features

  • •Event-driven, non-blocking I/O model making it efficient and lightweight
  • •Massive ecosystem with npm, the world's largest package registry
  • •Fast execution using Google's V8 JavaScript engine
  • •Single-threaded but highly scalable using event loop

Getting Started

Download Node.js from nodejs.org or use a version manager like `nvm`. Verify installation:

Check Node.js Version
node --version

To start a new project:

Create New Project
mkdir my-app
cd my-app
npm init -y

Your First Node.js Script

Create a file called hello.js and add this code:

hello.js
// hello.js - Your first Node.js script
console.log("Hello, World!");


// Use readline for user input
const readline = require('readline');
const rl = readline.createInterface({
    input: process.stdin,
    output: process.stdout
});


rl.question("What is your name? ", (name) => {
    console.log(`Nice to meet you, ${name}!`);
    rl.close();
});

Run it with:

Run Script
node hello.js

Build a Simple Web Server

Create server.js - your first web server in just 10 lines:

server.js
// server.js - A simple HTTP server
const http = require('http');


const server = http.createServer((req, res) => {
    res.writeHead(200, { 'Content-Type': 'text/html' });
    res.end('<h1>Hello from Node.js!</h1>');
});


server.listen(3000, () => {
    console.log('Server running at http://localhost:3000');
});

Run it and open http://localhost:3000 in your browser:

Run Server
node server.js

Node.js vs Bun: Which to Choose?

Choose Node.js if...

  • You need maximum stability and ecosystem compatibility
  • Your project requires specific Node.js-only packages
  • You're learning and want more tutorials/resources

Choose Bun if...

  • You want faster package installation and startup
  • You prefer native TypeScript support
  • You want an all-in-one tool (runtime + bundler + test runner)

Node.js / Bun: express

Fast, unopinionated, minimalist web framework for Node.js

Using npm (Node.js):
npm install express
Using bun:
bun add express
Usage in your code:
const lib = require('express'); // or import lib from 'express';

💡 Tips:

  • • Use npm init -y to initialize a new project.
  • • Bun is a fast alternative to npm/yarn/pnpm.
  • • Check package.json for script configurations.
