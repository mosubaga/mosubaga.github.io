Understanding Maximum Likelihood Estimation in Logistic Regression

Maximum Likelihood Estimation (MLE) is the fundamental procedure used to determine the parameters, the weights and biases, of a logistic regression model. While linear regression often relies on minimizing the sum of squares, logistic regression uses MLE because binary classification outcomes follow a distribution (Bernoulli) that is far from the Gaussian distribution assumed in standard least-squares models.

1. The Objective: Maximizing Probability

The core philosophy of MLE is to find the parameter values that make the observed data most probable. In a logistic regression setting, the model predicts the probability \( y \) that a given input belongs to a specific class (e.g., class 1). For every data point in the training set, we want the model to assign a high probability to the correct label: a probability close to 1 for positive instances and close to 0 for negative instances.

The model does not try to make every prediction exactly 0 or 1. Instead, it gives a probability \( y_n \in (0,1) \), then scores that probability against the observed Bernoulli outcome. A positive point contributes \( y_n \) to the likelihood, while a negative point contributes \( 1-y_n \). A good model therefore makes the correct class probable without leaving the probability scale.

2. Constructing the Likelihood Function

To evaluate a set of weights, we calculate the likelihood function, which represents the joint probability of all observed data points given those weights. Assuming the data points are independent and identically distributed (i.i.d.), this joint probability is calculated by multiplying the individual probabilities assigned by the model to each observed outcome.

For a binary dataset with labels \( t \in \{0, 1\} \), the likelihood function is expressed as:

$$ p(t \mid w) = \prod_{n} y_n^{t_n} (1 - y_n)^{1 - t_n} $$

In this formula, \( y_n \) is the model's predicted probability for the \( n \)-th instance, and \( t_n \) is the actual observed label.

For logistic regression, the predicted probability comes from the sigmoid function applied to a linear score:

$$ y_n = \sigma(a_n) = \frac{1}{1+\exp(-a_n)}, \qquad a_n = w^\top x_n $$

The linear score \( a_n \) can be any real number, but the sigmoid converts it into a valid probability. The intercept term is usually included by appending a constant feature \( x_{n0}=1 \).

Example (tiny dataset): Suppose the model predicts probabilities \( y = [0.9, 0.4, 0.7] \) for three examples, and the true labels are \( t = [1, 0, 1] \). The likelihood is:

$$ p(t \mid w) = 0.9^{1}(1-0.9)^{0} \cdot 0.4^{0}(1-0.4)^{1} \cdot 0.7^{1}(1-0.7)^{0} $$

So the joint probability is \( 0.9 \times 0.6 \times 0.7 = 0.378 \). A different set of weights would change the \( y_n \) values and therefore change this likelihood.

3. The Log-Likelihood and Numerical Stability

In practice, it is mathematically more convenient and numerically stable to maximize the logarithm of the likelihood function. This transformation is beneficial for several reasons:

  • From Products to Sums: The natural logarithm turns the product of many small probabilities into a sum of log-probabilities, simplifying the calculation of derivatives.
  • Numerical Underflow: Multiplying hundreds of probabilities (each between 0 and 1) can result in numbers so small they exceed a computer's precision; addition avoids this "underflow" issue.
  • Monotonicity: Because the logarithm is a monotonically increasing function, the parameters that maximize the log-likelihood are identical to those that maximize the original likelihood.

4. The Cross-Entropy Error Function

Machine learning frameworks typically use optimization algorithms designed to minimize a function rather than maximize it. Therefore, instead of maximizing the log-likelihood, we minimize the negative log-likelihood, which is known in logistic regression as the cross-entropy error function.

$$ E(w) = - \sum_{n} \left[t_n \ln(y_n) + (1 - t_n) \ln(1 - y_n)\right] $$

This error function is convex, meaning optimization algorithms do not have to navigate a landscape of competing local minima.

More precisely, the unregularized logistic regression loss is strictly convex when the feature matrix has enough independent information to identify the weights uniquely; otherwise, multiple weight vectors can give the same optimum.

Example (same dataset): Using \( y = [0.9, 0.4, 0.7] \) and \( t = [1, 0, 1] \):

$$ E(w) = -\left[\ln(0.9) + \ln(0.6) + \ln(0.7)\right] \approx 0.971 $$

Lower values of \( E(w) \) indicate a better fit to the observed labels.

5. Iterative Optimization

Unlike linear regression, which has a "closed-form" solution (a single formula to find the answer), logistic regression requires iterative numerical methods. Because the relationship between the weights and the output is nonlinear (due to the sigmoid function), the optimal weights are found by repeatedly adjusting them in the direction that reduces the cross-entropy error. Common methods include:

  • Gradient Descent: Repeatedly nudging the weights based on the gradient of the error.
  • Iterative Reweighted Least Squares (IRLS): A more advanced method using the Newton-Raphson scheme to reach the minimum more efficiently by using second-order information (the Hessian).

Example (one gradient step): With feature vector \( x = [1, x_1] \) and weights \( w = [b, w_1] \), the prediction is \( y = \sigma(w^\top x) \). A single gradient descent update looks like:

$$ w \leftarrow w - \eta \sum_{n} (y_n - t_n) x_n $$

Here \( \eta \) is the learning rate. If the model predicts too high when the true label is 0, the gradient term is positive and the weights are nudged downward.

Gradient

$$ \nabla E(w) = \sum_n (y_n - t_n)x_n $$

Hessian

$$ H = \sum_n y_n(1-y_n)x_nx_n^\top $$

The Hessian is positive semidefinite because each term is weighted by \( y_n(1-y_n) \ge 0 \). This is the algebraic reason logistic regression has a well-behaved convex optimization landscape under ordinary non-separable conditions.

6. Challenges: Linear Separability and Overfitting

A specific vulnerability of MLE in logistic regression occurs when a dataset is linearly separable (the two classes can be perfectly split by a straight line). In this case, MLE will attempt to drive the predicted probabilities for every training point to exactly 1 or 0. This causes the magnitude of the weights to go to infinity, turning the smooth S-curve of the sigmoid into an infinitely steep step function. To prevent this, practitioners often use regularization or Bayesian priors to keep the weights stable and bounded.

Example (regularization): A common fix is L2 regularization, which adds a penalty for large weights:

$$ E_{\text{reg}}(w) = E(w) + \lambda \lVert w \rVert^2 $$

The hyperparameter \( \lambda \) controls how strongly we discourage large weights, preventing runaway solutions on separable data.

7. A Compact Worked Example

Suppose we have a single feature \( x \) (plus a bias) and three observations:

  • Inputs: \( x = [0, 1, 2] \)
  • Labels: \( t = [0, 0, 1] \)

Let the model be \( y_n = \sigma(b + w x_n) \) with \( b = -1 \) and \( w = 1 \). Then:

$$ y = [\sigma(-1), \sigma(0), \sigma(1)] \approx [0.269, 0.5, 0.731] $$

The log-likelihood encourages the model to push \( y_1 \) and \( y_2 \) closer to 0, and \( y_3 \) closer to 1. After a few gradient steps, the model learns a steeper slope \( w \) and a more negative bias \( b \), which increases class separation while still keeping probabilities between 0 and 1.

8. Reading the Fitted Coefficients

Logistic regression coefficients are easiest to interpret on the log-odds scale. The model says:

$$ \log\left(\frac{y_n}{1-y_n}\right) = w^\top x_n $$

A one-unit increase in feature \(x_j\), holding the other features fixed, changes the log-odds by \(w_j\). Exponentiating gives an odds ratio:

$$ \frac{\text{new odds}}{\text{old odds}} = \exp(w_j) $$

This interpretation is one reason logistic regression remains useful even when more flexible classifiers are available: the fitted parameters are not just a decision rule, but a probability model with interpretable effects.

9. MLE as a Probability Calibration Problem

MLE rewards probability estimates that are confident when correct and cautious when the data are ambiguous. A prediction of \(0.99\) for an event that actually occurs receives a small loss, but the same \(0.99\) prediction for an event that does not occur receives a very large penalty. This asymmetry is exactly what makes cross-entropy sensitive to overconfident mistakes.

Key point: Logistic regression is not merely fitting a boundary. It is fitting the probabilities around that boundary. The decision boundary \(w^\top x = 0\) is where \(y=0.5\), but the likelihood also cares about how quickly the probabilities move away from 0.5 on either side.
