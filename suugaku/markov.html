Stochastic Processes

Markov Chains

A Markov chain is a stochastic process whose future depends only on the present state, not on how it got there. That single assumption — the Markov property — makes an enormous class of real-world systems tractable.

Overview

A discrete-time Markov chain is a sequence of random variables \(X_0, X_1, X_2, \ldots\) taking values in a countable state space \(\mathcal{S}\) and satisfying

\[ P(X_{n+1} = j \mid X_n = i,\, X_{n-1}, \ldots, X_0) = P(X_{n+1} = j \mid X_n = i). \]

The right-hand side, called the transition probability from state \(i\) to state \(j\), is denoted \(p_{ij}\). When these probabilities do not depend on time \(n\) the chain is time-homogeneous, which is the standard case.

1. Transition Matrix

For a finite state space with \(N\) states, all transition probabilities are collected into the transition matrix \(P\):

\[ P = (p_{ij})_{i,j \in \mathcal{S}}, \qquad p_{ij} \ge 0, \qquad \sum_{j \in \mathcal{S}} p_{ij} = 1 \text{ for all } i. \]

Each row sums to 1 because the chain must move somewhere. The probability of being in state \(j\) at time \(n\) given the initial distribution \(\pi^{(0)}\) is

\[ \pi^{(n)} = \pi^{(0)} P^n. \]

Higher powers \(P^n\) describe multi-step transitions: the \((i,j)\) entry of \(P^n\) is the probability of reaching \(j\) from \(i\) in exactly \(n\) steps.

2. A Worked Example — Weather Model

Suppose weather is either Sunny (S) or Rainy (R). Historical data gives the transition matrix

\[ P = \begin{pmatrix} 0.8 & 0.2 \\ 0.4 & 0.6 \end{pmatrix}, \]

where rows are today's state and columns are tomorrow's. If today is Sunny, then \(P(\text{Sunny tomorrow}) = 0.8\) and \(P(\text{Rainy tomorrow}) = 0.2\).

Starting from the distribution \(\pi^{(0)} = (1, 0)\) (definitely Sunny today), after two days:

\[ \pi^{(2)} = \pi^{(0)} P^2 = (1,0) \begin{pmatrix} 0.72 & 0.28 \\ 0.56 & 0.44 \end{pmatrix} = (0.72,\; 0.28). \]

There is a 72 % chance of sunshine two days from now.

3. Stationary Distribution

A probability vector \(\pi\) is a stationary distribution if it is unchanged by one step of the chain:

\[ \pi P = \pi, \qquad \sum_{i} \pi_i = 1, \qquad \pi_i \ge 0. \]

For the weather model above, solving \(\pi P = \pi\) gives \(\pi = (2/3,\; 1/3)\): in the long run it is sunny two-thirds of the time.

The Perron–Frobenius theorem guarantees that an irreducible, aperiodic chain on a finite state space has a unique stationary distribution, and the chain converges to it from any starting point.

4. Key Structural Properties

Irreducibility

Every state can be reached from every other state. Guarantees a unique stationary distribution.

Aperiodicity

The GCD of return times to any state is 1. Prevents oscillatory behavior and ensures convergence.

Recurrence

The chain returns to every state with probability 1. Positive recurrence additionally requires a finite mean return time.

Absorbing States

State \(i\) is absorbing if \(p_{ii}=1\). Once entered, the chain never leaves. Relevant in gambler's ruin and reliability models.

Detailed Balance and Reversibility

A chain satisfies detailed balance with respect to \(\pi\) when

\[ \pi_i \, p_{ij} = \pi_j \, p_{ji} \quad \text{for all } i, j. \]

This condition is stronger than stationarity but easier to verify. It says the probability flux from \(i\) to \(j\) equals the flux from \(j\) to \(i\) in steady state. Chains satisfying detailed balance are called reversible.

Why this matters for MCMC. The Metropolis–Hastings algorithm constructs a Markov chain whose stationary distribution is an arbitrary target distribution \(\pi\). It does so by designing transition probabilities that satisfy detailed balance — guaranteeing convergence to the correct distribution even when direct sampling is intractable.

5. Real-World Applications

The simplicity of the Markov property makes it surprisingly powerful across many fields.

  • PageRank. Google's original algorithm models web surfing as a Markov chain on the graph of web pages. The stationary distribution assigns each page a rank proportional to how often a random surfer visits it.
  • Natural Language Processing. A bigram language model is a Markov chain on words: the next word depends only on the current word. Text generation, spelling correction, and speech recognition all use Markov models.
  • Queueing Theory. Arrivals and service completions in a queue form a Markov chain on the number of customers. The M/M/1 queue is a classic example whose stationary distribution is geometric.
  • Finance. Credit-rating transitions (AAA → AA, etc.) are modeled as Markov chains. Default risk over a multi-year horizon is computed from the matrix power \(P^n\).
  • Biology. DNA sequence evolution is modeled as a continuous-time Markov chain on the four nucleotides. Phylogenetic trees are inferred by fitting transition rates to observed sequences.
  • Reinforcement Learning. An agent's interaction with an environment is formalized as a Markov Decision Process (MDP), an extension of Markov chains where the transition probabilities depend on the agent's action.

Continuous Time and Beyond

The continuous-time analog replaces the transition matrix with a generator matrix \(Q\), where off-diagonal entries are non-negative transition rates and each row sums to zero:

\[ P(t) = e^{Qt}, \qquad \frac{d}{dt}P(t) = P(t)Q. \]

The stationary distribution satisfies \(\pi Q = 0\). Continuous-time chains underlie chemical reaction networks, queueing models (Poisson arrivals), and the Gillespie algorithm used in computational biology.

Hidden Markov Models (HMMs) extend the framework further: the state is not observed directly; only noisy emissions from each state are seen. HMMs are the backbone of speech recognition, biological sequence analysis, and gesture detection.
