Learn MySQL & PostgreSQL
on Linux

A practical guide to installing, configuring and using the two most popular open-source relational databases — with commands you can copy and run in your own terminal.

A relational database stores data in tables. Keys link rows in one table to rows in another — that's the "relational" part. The moving dots are queries following those relationships.

MySQLWhat is MySQL?

MySQL is an open-source relational database management system (RDBMS) first released in 1995. It is famous for powering a huge share of the web — WordPress, many PHP applications, and large platforms such as YouTube and Facebook started on it. It is known for being easy to set up, fast for read-heavy web workloads, and extremely well documented because so many people use it.

On Linux you install it from your distribution's package manager, manage it with systemctl, and talk to it with the mysql command-line client.

PostgreSQLWhat is PostgreSQL?

PostgreSQL (often shortened to "Postgres") is an open-source RDBMS with roots going back to the 1980s at UC Berkeley. It has a reputation for correctness, strict adherence to the SQL standard, and advanced features: rich data types, powerful JSON support, full-text search, and an extension system (PostGIS for geographic data is a famous example).

On Linux it installs the same way — through the package manager — but it has a few of its own conventions, such as a dedicated postgres Linux user and the psql client with its backslash "meta-commands".

Why learn both?

In real jobs you rarely get to choose: one project uses MySQL, the next uses PostgreSQL. The core SQL is almost identical, but day-to-day administration — connecting, creating users, granting permissions, backing up — differs in small, annoying ways. Knowing both means:

  • You can move between projects and teams without friction.
  • You understand which differences are SQL-standard and which are vendor quirks.
  • You can make an informed choice when you do get to pick.
  • As a QA or automation engineer, you can set up test databases on any stack.

This site teaches the two side by side, so every task you learn for one database comes with the equivalent command for the other.

What this website covers

Installation & service management

Install MySQL and PostgreSQL on Ubuntu/Debian and Fedora/RHEL, then start, stop and check the services with systemd.

Running SQL files

Execute .sql scripts from the shell or from inside the client, and fix the most common errors when a script fails.

Users, roles & permissions

Create application users, grant only the privileges they need, and keep root/postgres accounts out of your apps.

Backup, restore & troubleshooting

Dump and restore databases with mysqldump and pg_dump, and diagnose services that won't start, auth failures and port conflicts.

Recommended learning path

Work through the steps in order — each page builds on the previous one.

  1. Install the database server

    Use your package manager to install MySQL or PostgreSQL (or both — they coexist happily).

  2. Start and stop the service

    Learn the systemctl commands to control and inspect the server process.

  3. Connect using the command line

    Open a session with the mysql or psql client.

  4. Create databases

    Use CREATE DATABASE and learn how to list and switch between databases.

  5. Run SQL files

    Execute schema and data scripts — see Running SQL files.

  6. Create users and manage permissions

    Set up least-privilege application users — see Users & permissions.

  7. Backup and restore databases

    Practise dumping and restoring before you need it — see Backup & restore.

  8. Troubleshoot common problems

    Learn to read service logs and fix auth errors — see Troubleshooting.

Tip — practise on a throwaway machine The best way to learn is a virtual machine or a cloud VM you can break without consequences. Every command on this site is safe to run on a fresh test system.
