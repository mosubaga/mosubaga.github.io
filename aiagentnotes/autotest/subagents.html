What Subagents Can We Build?

One prompt trying to be "test writer, selector fixer, bug hunter, and code reviewer" at once tends to blur its own rules — a fixer that also rewrites assertions to match a bug instead of flagging it, for example. Splitting the work into five focused subagents, each with a narrow job and its own system prompt, keeps every role honest.

Five Subagents

Generator

Given a page URL or a user story, drives Playwright MCP to inspect the live page and writes a new *.spec.ts file grounded in the real accessibility tree.

Fixer

Given a failing test and its error trace, re-inspects the page and patches the locator — it is never allowed to loosen or delete the assertion it's failing.

Bug Hunter

Crawls the live site looking for console errors, broken links, and layout breaks, and files a bug report — it does not write or touch test files at all.

Reviewer

Reads a generated or patched spec file for anti-patterns — hard-coded waits, brittle CSS selectors, missing await — before it's allowed into a PR.

Flake Triager

Reruns a failing test several times and classifies it as flaky (timing/network) vs. a real regression, so the Fixer and Bug Hunter aren't chasing noise.

Why Not One Big Prompt? Isolation

The Fixer's job depends on treating the assertion as fixed and only the locator as broken — a single prompt asked to "make the test pass" will happily do the easy thing and weaken the assertion instead. Keeping Fixer and Reviewer as separate subagents with separate contexts means the Reviewer catches that weakening even if the Fixer's prompt were ever loosened by mistake. More on why this matters for a small local model on the Context Engineering page.

Building Subagents as a Registry TypeScript · Ollama

Each subagent is a name, a system prompt, and the set of MCP tools it's allowed to call. A small model is fine for the Flake Triager; the Generator and Fixer benefit from a stronger one.

subagents.ts 
import ollama from 'ollama';

interface Subagent {
  model: string;
  system: string;
  allowedTools: string[];
}

const SUBAGENTS: Record<string, Subagent> = {
  generator: {
    model: 'qwen2.5:7b',
    system:
      'You write Playwright TypeScript tests. Always call browser_snapshot before ' +
      'writing a locator. Never invent a selector you have not seen in a snapshot.',
    allowedTools: ['browser_navigate', 'browser_snapshot', 'write_file'],
  },
  fixer: {
    model: 'qwen2.5:7b',
    system:
      'A Playwright test is failing. Re-inspect the live page and fix the locator ' +
      'only. Do not change, remove, or loosen the assertion being tested.',
    allowedTools: ['browser_navigate', 'browser_snapshot', 'read_file', 'write_file'],
  },
  bugHunter: {
    model: 'qwen2.5:7b',
    system:
      'Explore the site. Report console errors, broken links, and layout issues as ' +
      'a short bug list. You do not write or edit test files.',
    allowedTools: ['browser_navigate', 'browser_console_messages', 'browser_snapshot'],
  },
  reviewer: {
    model: 'qwen2.5:3b',
    system:
      'Review a Playwright spec file for anti-patterns: hard-coded waits, brittle ' +
      'CSS selectors, missing awaits. List problems, do not rewrite the file.',
    allowedTools: ['read_file'],
  },
  flakeTriager: {
    model: 'qwen2.5:3b',
    system:
      'Given N rerun results for one test, classify it as FLAKY or REGRESSION and ' +
      'explain the pattern in one sentence.',
    allowedTools: [],
  },
};

async function runSubagent(name: string, userContent: string): Promise<string> {
  const agent = SUBAGENTS[name];
  const response = await ollama.chat({
    model: agent.model,
    messages: [
      { role: 'system', content: agent.system },
      { role: 'user', content: userContent },
    ],
  });
  return response.message.content;
}

A Small Orchestrator TypeScript

A thin script — the same one triggered by the loop described on the Loop Engineering page — routes between subagents for one CI run.

run_ci_cycle.ts 
import { runSubagent } from './subagents';
import { runTests, type TestFailure } from './test-runner';

async function handleFailure(failure: TestFailure): Promise<void> {
  const verdict = await runSubagent(
    'flakeTriager',
    `Test: ${failure.title}\nReruns: ${JSON.stringify(failure.rerunResults)}`
  );

  if (verdict.includes('FLAKY')) {
    console.log(`Skipping ${failure.title} — classified as flaky`);
    return;
  }

  // Real regression — hand off to the Fixer, then have the Reviewer sanity-check the patch.
  const patch = await runSubagent('fixer', `File: ${failure.file}\nError: ${failure.error}`);
  const review = await runSubagent('reviewer', patch);
  console.log({ file: failure.file, patch, review });
}

const results = await runTests();
for (const failure of results.failures) {
  await handleFailure(failure);
}
