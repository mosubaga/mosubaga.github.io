What Other AI Agents Can I Use?

The custom Ollama + Playwright MCP harness on the setup page is one option — a fully scriptable, fully local agent for CI. But day-to-day, an interactive coding agent already sitting in your editor can drive the same MCP tools while a person watches. Here's how the main ones compare for this job.

Coding Agents That Can Drive Playwright MCP Comparison

Agent MCP support Local LLM (Ollama) Best fit here
Claude Code Native, first-class Not built-in Interactively driving Playwright MCP and other servers straight from the terminal — this site was built with it.
Cursor Native (mcp.json) Via OpenAI-compatible endpoint Editor-embedded generation with the same MCP config shown on the setup page.
GitHub Copilot (agent mode) Native Not built-in Teams already standardized on GitHub — same MCP servers, PR review baked into the platform.
OpenAI Codex CLI Native Not built-in Scripted, terminal-first runs similar in spirit to the custom harness, without writing the harness.
Continue.dev Native First-class, built for it Fully open-source and local-first — closest match to the "no cloud API" assumption on this site.
Cline (VS Code) Native, can generate new MCP servers on request First-class A good middle ground — a full VS Code agent that talks to Ollama and can even write the coverage server from the MCP page for you.

The Payoff: One Config, Many Hosts Portability

Because Playwright MCP and the custom coverage server are MCP servers, the server command and arguments from the setup page carry across these hosts. Each host stores that configuration in a different place, and Continue's primary format is YAML, so check the host's documentation when moving it. Building on MCP still avoids rewriting the server itself. 

// JSON-style MCP configuration used by several hosts; file location varies
{
  "mcpServers": {
    "playwright": { "command": "npx", "args": ["@playwright/mcp@latest", "--headless"] }
  }
}

Interactive Agent vs. the CI Harness When to Use Which

Use an editor agent (Cline, Continue.dev, Cursor, Claude Code) while a developer is actively writing or debugging a test — a person is right there to catch a bad suggestion. Use the custom Ollama + MCP harness from the setup page for the unattended parts — the nightly loop on the Loop Engineering page — where nobody is watching in real time and the guardrails have to do that job instead.
