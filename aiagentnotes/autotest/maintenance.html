How AI Maintains Tests Long-Term

Writing the first version of a test suite is the easy part. Keeping it passing, relevant, and honest as mosubaga.github.io changes over months is where most hand-written suites rot. This is where every piece from the earlier pages — subagents, context budgets, the bounded loop — comes together into ongoing upkeep.

Four Ongoing Jobs

Self-Healing Selectors

When a locator starts timing out, the Fixer subagent re-snapshots the live page, finds the closest matching element, and proposes the updated locator — the bounded loop from the Loop Engineering page drives this end to end.

Bug-Hunting Crawls

The Bug Hunter subagent walks every page, capturing console errors, broken links, and a11y violations via the a11y-scan server from the MCP page — and only ever files a report, never edits a test.

Flaky Test Detection

The Flake Triager reruns a failure a few times before anyone touches code, so a timing hiccup doesn't get "fixed" by loosening an assertion that was actually correct.

Coverage Gap-Finding

The custom coverage server from the MCP page diffs the site's routes against existing spec files, so the Generator subagent gets pointed at real gaps instead of re-testing the homepage for the tenth time.

Maintenance Task → Who Handles It Reference

Task Subagent Tools it calls
A locator stopped matching after a UI change Fixer browser_snapshot, write_file
A patch needs a second opinion before merging Reviewer read_file
A test fails intermittently, not consistently Flake Triager none — reasons over rerun logs only
The live site has a console error or broken link Bug Hunter browser_navigate, browser_console_messages
A new page shipped with no test at all Generator list_untested_routes, browser_snapshot, write_file

The One Rule That Ties It All Together Human-in-the-Loop

None of these subagents merge their own work. Every self-healed selector, every new spec file, every bug report lands as a draft PR — with the agent's reasoning in the description, per the guardrails on the Loop Engineering page — so a person makes the last call on whether the agent actually understood what changed, or just found a way to make the test pass.
