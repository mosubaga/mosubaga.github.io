Retrieval-Augmented Generation (RAG)

A technique that gives an LLM access to external knowledge at inference time — no retraining required. The model retrieves relevant facts, augments its context with them, and generates a grounded, accurate response.

What is RAG? Definition

LLMs only "know" what was in their training data. RAG bridges that gap by connecting them to live, domain-specific sources at query time — no fine-tuning needed.

Latest Information Private Documents Technical Manuals

Key distinction: RAG is referencing, not learning. Instead of training the model to memorise facts (fine-tuning), RAG hands the model a "book" to consult while answering.

The Standard RAG Workflow

1

User Input — The user asks a question.

2

Query Extraction — The LLM analyses the question to decide if external info is needed and generates search keywords.

3

Retrieval — The app searches an external database (Vector DB or full-text search) using those keywords.

4

Data Acquisition — Relevant text snippets are fetched from the database.

5

Augmentation — The retrieved snippets are injected into the LLM's context window as background information.

6

Request — A final prompt containing both the question and the retrieved evidence is sent to the LLM.

7

Generation — The LLM produces a grounded response based on the provided evidence.

Step 2 — Basic Implementation Python

This script simulates retrieval by scanning a local knowledge base, then passes the relevant snippet to the LLM as context before generating the answer.

simple_rag.py 
from openai import OpenAI
import os

# 1. Initialize the client — key is read from the environment
client = OpenAI(api_key=os.environ.get("OPENAI_API_KEY"))

# 2. In-memory knowledge base (use a Vector DB in production)
knowledge_base = {
    "context_engineering": "Context Engineering is the art of deciding what info to give or discard in an LLM's context window.",
    "mcp": "MCP (Model Context Protocol) is an open standard by Anthropic for connecting AI apps to external tools."
}

def simple_rag_app(user_query: str) -> str:
    # 3. Retrieval — keyword match (replace with Vector Search in production)
    retrieved_info = ""
    if "context engineering" in user_query.lower():
        retrieved_info = knowledge_base["context_engineering"]
    elif "mcp" in user_query.lower():
        retrieved_info = knowledge_base["mcp"]

    # 4. Augment: inject retrieved context into the system prompt
    messages = [
        {"role": "system", "content": f"Use this background info to answer accurately: {retrieved_info}"},
        {"role": "user",   "content": user_query}
    ]

    # 5. Generate
    response = client.chat.completions.create(
        model="gpt-4o",
        messages=messages
    )
    return response.choices[0].message.content  # [0] — first (and only) choice

print(simple_rag_app("What is Context Engineering?"))

Advanced Improvements for Production

The keyword match above is a starting point. Real-world RAG applies context engineering techniques to improve accuracy and scale.

Vector Search

Convert text into numeric vectors to find semantically similar content — even when the user's exact keywords don't appear in the document.

Reranking

After retrieval, use a cross-encoder model to re-score and re-order results so only the most relevant snippets are injected into the context window.

Hybrid Search

Combine keyword matching (BM25) with vector similarity to get the strengths of both — exact term hits plus semantic relevance.

Chunking

Split large PDFs or documents into smaller, meaningful pieces before indexing — so the model receives focused context rather than entire files.
