What Subagents Can We Build?

One giant prompt trying to be "textbook expert, problem writer, patient tutor, strict grader, and scheduler" all at once tends to blur its own rules — a hint slips into a full solution, or grading criteria get fuzzy. Splitting the work into five focused subagents, each with a narrow job and its own system prompt, keeps every role sharp.

Five Subagents

Problem-Setter

Takes retrieved textbook excerpts and today's syllabus topic, drafts a problem set with an answer key. Never sees a student's past answers — it only cares about the material.

Tutor

Engages a stuck student with graduated hints — nudge, then partial step, then full worked solution — and is instructed to never reveal the final answer on the first hint.

Grader

Checks a submitted answer, combining a deterministic sympy check for numeric/symbolic answers with an LLM judgment for free-form proof-style responses.

Scheduler

Decides which topic comes next based on the syllabus pace and which topics a student is weak in — mostly logic, with a small LLM call to break ties or re-sequence around a struggle.

Progress-Reporter

Summarizes a week of attempts into a short digest — streaks, accuracy by topic, hints used — instead of replaying the full history every time it's asked "how am I doing?"

Why Not One Big Prompt? Isolation

The Tutor's job depends on not knowing the answer key up front — otherwise it's tempted to just state it. The Grader's job depends on seeing the answer key. Keeping those in separate subagents with separate contexts isn't just tidier code, it's what makes the "give hints, don't give answers" rule actually hold. This is the same idea explored in more depth on the Context Engineering page.

Building Subagents as a Registry Python · Ollama

Each subagent is just a name, a system prompt, and (optionally) a different local model — a small model is often enough for the Scheduler, while the Tutor benefits from a stronger one.

subagents.py 
import ollama

SUBAGENTS = {
    "problem_setter": {
        "model": "qwen2.5:7b",
        "system": (
            "You write college-level practice problems strictly from the provided "
            "textbook excerpts. Include a full answer key. Never discuss a specific "
            "student's history."
        ),
    },
    "tutor": {
        "model": "qwen2.5:7b",
        "system": (
            "You are a patient tutor. A student is stuck on a problem. Give ONE small "
            "hint that nudges their thinking. Do not state the final answer, even if asked "
            "directly. Escalate to a bigger hint only after repeated wrong attempts."
        ),
    },
    "grader": {
        "model": "qwen2.5:7b",
        "system": (
            "You check a student's answer against the provided answer key and a "
            "symbolic-check result. Explain briefly why the answer is correct or "
            "incorrect. Be concise and specific."
        ),
    },
    "scheduler": {
        "model": "qwen2.5:3b",
        "system": (
            "Given the syllabus order and a list of the student's weak topics, choose "
            "tomorrow's topic. Prefer syllabus order unless a weak topic needs review first."
        ),
    },
    "progress_reporter": {
        "model": "qwen2.5:3b",
        "system": (
            "Summarize a week of problem attempts into 3-4 sentences: accuracy trend, "
            "weakest topic, and one encouraging note."
        ),
    },
}

def run_subagent(name: str, user_content: str) -> str:
    agent = SUBAGENTS[name]
    response = ollama.chat(
        model=agent["model"],
        messages=[
            {"role": "system", "content": agent["system"]},
            {"role": "user", "content": user_content},
        ],
    )
    return response["message"]["content"]

hint = run_subagent(
    "tutor",
    "Problem: Find the eigenvalues of [[2,1],[1,2]]. "
    "Student tried: they set up det(A) = 0 instead of det(A - λI) = 0.",
)
print(hint)

The Daily Orchestrator Python

A thin script — triggered by cron, see Context Engineering — routes between subagents and MCP tools for one student's daily cycle.

daily_job.py 
from subagents import run_subagent
from generate_problem_set import retrieve

def run_daily_cycle(student_id: str, subject: str, topic: str) -> dict:
    # 1. Scheduler already chose `topic` for today (not shown here)
    # 2. Ground the problem set in real textbook content
    excerpts = retrieve(subject, topic)
    context = "\n\n".join(e["text"] for e in excerpts)

    # 3. Problem-Setter drafts today's set
    problem_set = run_subagent(
        "problem_setter", f"Topic: {topic}\n\nExcerpts:\n{context}"
    )

    return {"student_id": student_id, "subject": subject, "topic": topic, "problem_set": problem_set}

def handle_submission(problem: str, student_answer: str, is_correct: bool) -> str:
    if is_correct:
        return run_subagent("grader", f"Problem: {problem}\nAnswer: {student_answer}\nResult: correct")
    # Wrong answer — hand off to the Tutor instead of the Grader explaining the fix
    return run_subagent("tutor", f"Problem: {problem}\nStudent tried: {student_answer}")
