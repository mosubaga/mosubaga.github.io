Can RAG Build Knowledge From the PDF Textbooks?

Yes. The agent must never invent a theorem or formula — every problem it writes has to trace back to the actual Calculus, Linear Algebra, or Probability/Statistics textbook the student owns. RAG is exactly the tool for that: it lets a local Ollama model consult the real textbook instead of relying on whatever it half-remembers from training.

Why RAG Is the Right Fit Rationale

A general-purpose LLM already "knows" calculus in a fuzzy, statistical sense — but it doesn't know your textbook's notation, numbering, or the specific subset of material covered up to Chapter 6. RAG closes that gap without fine-tuning:

Matches the syllabus Cites page & chapter Works offline, per-book

Key idea: instead of asking Ollama "write a calculus problem," the agent asks it to "write a problem using this excerpt from Chapter 4, Section 2" — retrieval turns a vague request into a grounded one.

The Textbook RAG Workflow

Extract — pull raw text page-by-page from each subject's PDF.

Chunk — split into small sections, tagged with subject, chapter, and page.

Embed — turn each chunk into a vector using an Ollama embedding model.

Store — persist the vectors in a local Chroma collection, one per subject.

Retrieve — each morning, fetch the chunks matching today's syllabus topic.

Generate — hand the retrieved excerpts to Ollama to draft a grounded problem set.

Step 1 — Ingest the Textbooks Python

Run this once per textbook (or whenever a new edition arrives). It extracts, chunks, embeds with a local Ollama embedding model, and writes everything into a persistent Chroma collection.

ingest_textbooks.py 
# Requires: ollama serve && ollama pull nomic-embed-text
# pip install pdfplumber chromadb ollama
import pdfplumber
import chromadb
import ollama

TEXTBOOKS = {
    "calculus": "textbooks/calculus.pdf",
    "linear_algebra": "textbooks/linear_algebra.pdf",
    "probability_statistics": "textbooks/prob_stats.pdf",
}
EMBED_MODEL = "nomic-embed-text"
CHUNK_WORDS = 180

def chunk_page(text: str, size: int = CHUNK_WORDS) -> list[str]:
    # Split on blank lines first so a proof/example isn't cut mid-formula,
    # then fall back to a word-count window for long paragraphs.
    paragraphs = [p.strip() for p in text.split("\n\n") if p.strip()]
    chunks, buf = [], []
    for para in paragraphs:
        buf.append(para)
        if sum(len(p.split()) for p in buf) >= size:
            chunks.append("\n\n".join(buf))
            buf = []
    if buf:
        chunks.append("\n\n".join(buf))
    return chunks

client = chromadb.PersistentClient(path="./vector_store")
collection = client.get_or_create_collection("textbooks")

for subject, path in TEXTBOOKS.items():
    with pdfplumber.open(path) as pdf:
        for page_num, page in enumerate(pdf.pages, start=1):
            text = page.extract_text() or ""
            for i, chunk in enumerate(chunk_page(text)):
                vector = ollama.embeddings(model=EMBED_MODEL, prompt=chunk)["embedding"]
                collection.add(
                    ids=[f"{subject}-p{page_num}-c{i}"],
                    embeddings=[vector],
                    documents=[chunk],
                    metadatas=[{"subject": subject, "page": page_num}],
                )
    print(f"Indexed {subject} from {path}")

Step 2 — Retrieve & Generate a Problem Set Python

This runs as part of the daily job: pull the excerpts relevant to today's topic, then ask the local chat model to write problems only from that material, with page citations for each one.

generate_problem_set.py 
# Requires: ollama pull qwen2.5:7b
import json
import chromadb
import ollama

client = chromadb.PersistentClient(path="./vector_store")
collection = client.get_collection("textbooks")
CHAT_MODEL = "qwen2.5:7b"

def retrieve(subject: str, topic: str, k: int = 5) -> list[dict]:
    result = collection.query(
        query_texts=[topic],
        n_results=k,
        where={"subject": subject},
    )
    return [
        {"text": doc, "page": meta["page"]}
        for doc, meta in zip(result["documents"][0], result["metadatas"][0])
    ]

def generate_problem_set(subject: str, topic: str, count: int = 4) -> dict:
    excerpts = retrieve(subject, topic)
    context = "\n\n".join(f"[p.{e['page']}] {e['text']}" for e in excerpts)

    system = (
        "You are a professor writing practice problems for college students. "
        "Use ONLY the textbook excerpts below as your source of facts and notation. "
        "Cite the page number each problem is drawn from. "
        'Reply as JSON: {"problems": [{"question": str, "page": int, "answer": str}]}'
    )
    user = f"Subject: {subject}\nTopic: {topic}\nWrite {count} problems.\n\nExcerpts:\n{context}"

    response = ollama.chat(
        model=CHAT_MODEL,
        messages=[
            {"role": "system", "content": system},
            {"role": "user", "content": user},
        ],
        format="json",
    )
    return json.loads(response["message"]["content"])

problem_set = generate_problem_set("linear_algebra", "eigenvalues and eigenvectors")
print(json.dumps(problem_set, indent=2))

Chunking Math Textbooks Well

Keep Proofs Whole

Split on paragraph boundaries, not a fixed character count — cutting a theorem away from its proof or a formula mid-derivation produces nonsense excerpts.

Tag by Chapter

Store subject, chapter, and page in metadata so retrieval can be filtered to "material covered so far" and every generated problem can cite its source.

Small Chunks

~150–200 words per chunk keeps retrieval precise — a whole chapter as one chunk buries the one relevant definition in irrelevant surrounding text.

Failure Modes to Watch For Math-Specific

Broken Notation

PDF text extraction can mangle superscripts, integrals, and Greek letters into garbled characters.

Scanned Pages

Older textbooks scanned as images yield no extractable text at all — these pages need OCR before chunking.

Split Theorems

A theorem statement and its proof landing in different chunks means neither retrieval hit is fully useful alone.

Table-Based Layouts

Statistics tables (z-scores, distributions) often extract in scrambled row/column order.
