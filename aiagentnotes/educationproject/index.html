A Study Agent That Never Runs Out of Problems

A walkthrough of how to build an AI Agent that reads a college textbook, generates a fresh problem set every day for a full semester, helps a student when they get stuck, and checks their work — built with a local Ollama LLM and Python.

Start with RAG See the MCP Servers

The Daily Pipeline

Calculus, Linear Algebra & Prob/Stats textbooks
Chunked & embedded into a local vector index
Ollama generates today's problem set
Student solves, checks, and asks for hints
Repeats daily, automatically, for 15 weeks

The Scenario

You're asked to build an AI Agent for college students studying Calculus, Linear Algebra, and Probability/Statistics. The agent must:

1

Start from the given textbooks in PDF format — no other source of truth.

2

Combine those textbooks with an LLM to generate problem sets grounded in the actual course material.

3

Let the student solve, self-check, and ask for help without simply handing over the answer.

4

Generate a new problem set every day, automatically, without a human pressing "go."

5

Keep running for an entire semester — roughly 15 weeks, ~100 class days.

Assumptions Constraints

  Local LLM via Ollama   Model already downloaded   Python throughout   No cloud API required

Four Questions, Four Building Blocks

Each page below answers one question about how this agent is actually built.

Q1 · Retrieval

Can RAG turn the PDFs into knowledge?

Yes — chunk the textbooks, embed them locally, and retrieve grounded excerpts before generating problems.

 Q2 · Protocol

What MCP servers does the agent need?

Five small servers — textbook search, problem-set generation, grading, scheduling, and progress tracking.

 Q3 · Delegation

What subagents can we build?

A problem-setter, tutor, grader, scheduler, and progress-reporter — each with a narrow job and prompt.

 Q4 · Memory

How does Context Engineering keep it sane?

Daily context resets and compact student digests stop a 15-week agent from drowning in its own history.

Tech Stack

LLM

Ollama running a locally downloaded model (e.g. qwen2.5:7b) for both chat and embeddings.

Retrieval

PDF parsing with pdfplumber, embeddings from Ollama, stored in a local Chroma vector index.

Tools

MCP servers built with FastMCP expose retrieval, generation, and grading as callable tools.

Automation

A cron job (or Python schedule) triggers the daily pipeline for the length of the semester.
