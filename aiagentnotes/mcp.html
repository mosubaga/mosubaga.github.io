Protocol Guide

Understanding the Model Context Protocol (MCP)

The Model Context Protocol (MCP), introduced by Anthropic in late 2024, is an open standard designed to provide a consistent, secure, and efficient way for AI systems to connect to external services, tools, and data sources. It acts as a universal bridge between LLMs and the wider digital world.

The "USB-C for AI" Open Standard

Before MCP, developers had to write bespoke "glue code" for every unique integration between an agent and a tool. MCP standardizes this connection: you build a tool server once, and any compatible client — like Claude Desktop, Cursor, or a custom AI agent — can automatically discover and use those capabilities without any extra wiring.

Why MCP is Critical for AI Agents

Interoperability

Instead of building separate integrations for every platform, MCP provides a single client-server contract that works across the entire AI ecosystem.

Dynamic Discovery

Agents can programmatically "list" available tools on a server, letting them understand at runtime what capabilities they have at their disposal — no hardcoding required.

Separation of Concerns

MCP separates the "brain" (model reasoning) from the "arms and legs" (tool execution), letting developers focus on building capable agents rather than maintaining brittle connectors.

Scalability

MCP lets agents treat local documents and remote databases with the same standardized interface, making it easy to swap or add new data sources as needs evolve.

Core Components of a Server

Every MCP server exposes functionality through three primary components:

Tools  (Verbs)

Executable functions that perform actions — fetching weather, sending an email, or querying a database.

Resources  (Nouns)

Read-only data sources that provide context to the model, such as static files, documentation, or database schemas.

Prompts  (Templates)

Predefined instructions that guide the LLM on how to interact with specialized capabilities exposed by the server.

How MCP Connects Architecture

MCP has three roles: the AI application is the host, the connector inside it is the client, and the external capability is exposed by the server.

stdio

Local MCP servers commonly run over stdio, where the AI client starts the server process and communicates through stdin/stdout.

HTTP

Remote MCP servers typically use HTTP-based transports, allowing agents to call tools hosted on external machines.

Config

Clients discover servers from a configuration file. Example for a local Python server:

{
  "mcpServers": {
    "research-assistant": {
      "command": "python",
      "args": ["research_server.py"]
    }
  }
}

MCP Safety Checklist Important

MCP standardizes how clients discover and call tools, but it does not remove the need for authentication, authorization, input validation, and user approval for sensitive actions.

  • Treat MCP servers as trusted software, not passive data files.
  • Use least-privilege credentials for each server.
  • Separate read-only tools from tools that modify files, databases, calendars, or money.
  • Show users what tool will run before sensitive actions.
  • Validate tool inputs and outputs with schemas.
  • Watch for prompt injection in tool descriptions, retrieved documents, and web pages.
  • Log tool calls so agent behavior can be audited later.

Building a Simple MCP Service in Python FastMCP

Using the FastMCP library, creating a service is straightforward. This example exposes a tool that returns a list of verified research sources. 

from mcp.server.fastmcp import FastMCP

# Initialize the MCP server with a descriptive name
mcp = FastMCP("Research Assistant Service")

# Use the @mcp.tool decorator to define actionable capabilities
@mcp.tool()
def get_research_sources() -> list[str]:
    """Provides a list of verified research sources for the agent."""
    return [
        "Wikipedia",
        "Google Scholar",
        "ArXiv",
        "PubMed"
    ]

if __name__ == "__main__":
    # Start the server using STDIO transport (standard for local agents)
    mcp.run(transport='stdio')

Install this server into Claude Desktop or Cursor and the AI will automatically see the get_research_sources tool and know exactly when to call it.
