Hands-On Guide

Building Your First AI Agent

An AI Agent is more than just a chatbot — it uses an LLM as its "brain" to autonomously plan, reason, and execute multi-step tasks. Unlike a standard chatbot, an agent runs a Sense-Plan-Act-Learn loop and interacts with the real world through tools.

The Core Formula Key Concept

Agent = LLM (Brain) + Tools (Action Space) + Loop (Execution Structure)

Example 1: Research Agent Python smolagents · Ollama

This example uses smolagents with a local Ollama + Qwen 2.5 model — no cloud API key required. The agent searches the web for financial data and summarizes it into a table. Local models are useful for privacy and experimentation, but tool-calling behavior depends heavily on the model size, quantization, prompt, and schema clarity.

research_agent.py 
# Requires: pip install smolagents litellm
# Requires Ollama running locally — start with: ollama serve
# Pull the model first:           ollama pull qwen2.5:7b
from smolagents import CodeAgent, DuckDuckGoSearchTool, LiteLLMModel

# 1. Point to the local Ollama + Qwen model (no cloud API key needed)
model = LiteLLMModel(
    model_id="ollama/qwen2.5:7b",
    api_base="http://localhost:11434"
)

# 2. Create the agent with a search tool
agent = CodeAgent(
    tools=[DuckDuckGoSearchTool()],
    model=model,
    add_base_tools=True
)

# 3. Assign a realistic research task
message = """
Find the Q4 2024 revenue for Amazon, Google, and Apple.
Compare them in a clear Markdown table.
"""

# The agent searches the web, finds the data, and generates the table.
agent.run(message)

What Makes This an Agent?

  • Goal: research Q4 revenue for three companies
  • Model: local Qwen 2.5 via Ollama
  • Tool: DuckDuckGo web search
  • Loop: CodeAgent runs plan-act-observe until satisfied
  • Final answer: a formatted Markdown table

If an example fails with a local model, try a stronger model, reduce the number of tools, or make tool descriptions more explicit.

Example 2: Minimal Local Tool-Using Agent Python Ollama · Raw Loop

This example avoids a heavy framework and shows the core agent loop explicitly: model picks a tool, the program runs it, and the model receives the result before writing the final answer.

local_agent.py 
# Requires: ollama serve && ollama pull qwen2.5:7b
# pip install ollama
import json
import ollama

TOOLS = [
    {
        "type": "function",
        "function": {
            "name": "calculate",
            "description": "Evaluate a simple arithmetic expression.",
            "parameters": {
                "type": "object",
                "properties": {
                    "expression": {
                        "type": "string",
                        "description": "Arithmetic expression such as '12 * 7 + 3'.",
                    }
                },
                "required": ["expression"],
            },
        },
    }
]

def calculate(expression: str) -> str:
    allowed = set("0123456789+-*/(). ")
    if not set(expression) <= allowed:
        return "Rejected: expression contains unsupported characters."
    return str(eval(expression, {"__builtins__": {}}, {}))

messages = [
    {"role": "system", "content": "Use tools when calculation is needed."},
    {"role": "user", "content": "What is (42 * 19) + 17?"},
]

response = ollama.chat(model="qwen2.5:7b", messages=messages, tools=TOOLS)
message = response["message"]
messages.append(message)

for call in message.get("tool_calls", []):
    if call["function"]["name"] == "calculate":
        args = call["function"]["arguments"]
        if isinstance(args, str):
            args = json.loads(args)
        result = calculate(args["expression"])
        messages.append({"role": "tool", "name": "calculate", "content": result})

final = ollama.chat(model="qwen2.5:7b", messages=messages)
print(final["message"]["content"])

What Makes This an Agent?

  • Goal: answer a calculation question
  • Model: local Qwen 2.5 via Ollama
  • Tool: sandboxed arithmetic evaluator
  • Loop: explicit — model calls tool, program executes, model sees result
  • Final answer: natural-language response with the computed value

Example 3: Browser-Based Assistant TypeScript @openai/agents

TypeScript agents run on Node.js using the official @openai/agents package. The SDK reads OPENAI_API_KEY automatically from the environment. Model names change; check provider docs before publishing.

market_assistant.ts 
// Requires: npm install @openai/agents
// API key:  export OPENAI_API_KEY="sk-..."
import { Agent, run, tool } from "@openai/agents";

const searchTool = tool({
  name: "web_search",
  description: "Searches for up-to-date market information",
  parameters: {
    type: "object",
    properties: {
      query: { type: "string", description: "The search query" }
    },
    required: ["query"]
  },
  execute: async ({ query }) => `Search results for: ${query}`
});

const agent = new Agent({
  name: "MarketResearcher",
  instructions: "You are a market analyst. Search for trends and summarize findings.",
  model: "choose a currently supported fast model",
  tools: [searchTool]
});

async function runResearch() {
  const result = await run(agent, "What are the top 3 AI trends in 2025?");
  console.log("Research Result:", result.finalOutput);
}

runResearch();

What Makes This an Agent?

  • Goal: identify top AI trends
  • Model: hosted LLM via OpenAI
  • Tool: web search function
  • Loop: SDK handles plan-act-observe internally
  • Final answer: structured summary in finalOutput

Example 4: Local Tool Call (TypeScript) TypeScript Ollama · Raw Loop

A fully local agent loop in Node.js — no cloud API required.

local_tool_agent.ts 
// Requires: ollama serve && ollama pull qwen2.5:7b
// npm install ollama
import ollama from "ollama";

const tools = [
  {
    type: "function",
    function: {
      name: "getProjectStatus",
      description: "Returns the current project status for a project ID.",
      parameters: {
        type: "object",
        properties: { projectId: { type: "string" } },
        required: ["projectId"],
      },
    },
  },
];

function getProjectStatus(projectId: string) {
  const data: Record<string, string> = {
    "agent-notes": "Draft complete. Needs citations and local LLM examples.",
  };
  return data[projectId] ?? "Unknown project.";
}

const messages: any[] = [
  { role: "system", content: "Use tools when project status is requested." },
  { role: "user", content: "What is the status of project agent-notes?" },
];

const first = await ollama.chat({ model: "qwen2.5:7b", messages, tools });
messages.push(first.message);

for (const call of first.message.tool_calls ?? []) {
  if (call.function.name === "getProjectStatus") {
    const args =
      typeof call.function.arguments === "string"
        ? JSON.parse(call.function.arguments)
        : call.function.arguments;
    messages.push({
      role: "tool",
      name: "getProjectStatus",
      content: getProjectStatus(args.projectId),
    });
  }
}

const final = await ollama.chat({ model: "qwen2.5:7b", messages });
console.log(final.message.content);

What Makes This an Agent?

  • Goal: retrieve project status in natural language
  • Model: local Qwen 2.5 via Ollama
  • Tool: getProjectStatus data lookup
  • Loop: explicit two-step — first call decides, second call answers
  • Final answer: conversational reply with retrieved status

Local LLM Tradeoffs Decision Guide

Advantages

  • Privacy — data stays on your machine
  • Cost control — no per-token billing
  • Offline development

Limitations

  • Slower inference on small machines
  • Weaker tool-following on smaller models
  • Quality depends on quantization and prompt

Why These Examples Matter

Autonomy

In both cases, the agent does not ask "Should I search?" — it understands the intent and executes immediately without per-step confirmation.

Cross-Platform

The Python ecosystem excels at data-heavy research, while TypeScript enables high-performance, real-time web UI agents — each ecosystem plays to its strengths.

Structured Data

Modern agents avoid brittle text parsing by returning typed data models — Pydantic in Python or JSON schemas in TypeScript — for reliable, machine-readable outputs.

What You Can Build Next

By moving beyond simple chat interfaces, you can build agents that serve as proactive "doers" — handling complex tasks like SEO automation, financial analysis, or technical documentation updates entirely on their own.
